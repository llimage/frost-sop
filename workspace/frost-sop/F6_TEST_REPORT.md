## 八、缺陷修复记录

### DEFECT-001（已修复）
- **严重级别**：高
- **现象**：孙辈Agent组装后 skills 为空，阶段执行产出为空字符串（静默失败）
- **根因**：`skills/llm.py` 的 `_mock_response_for_prompt()` 中，Agent组装prompt和Skill合成prompt被通用"需求"关键词误路由，返回了非JSON格式的内容，触发fallback配置 `sop_steps=[], skills={}`
- **修复**：
  1. 重构 `_mock_response_for_prompt()` 路由优先级，将Agent组装（`"分析以下Agent需求"`）、Skill合成（`"Skill设计助手"`）、Skill匹配、LLM产出生成四类高优先级 prompt 放到最前面，避免通用关键词误匹配
  2. 在 `skills/orchestration.py` 的 `execute_stage` 中增加显式检查：若组装后 `child.skills` 为空或 `sop_steps` 为空，立即返回失败结果，防止静默失败
- **验证**：E2E 7/7 通过，DQ 8/8 通过

### DEFECT-002（已修复）
- **严重级别**：中
- **现象**：`test_f6_parallel.py` 使用 `threading` + `patch_openai()` 在多线程环境下死锁，4个 PAR 用例均无法完成
- **根因**：`unittest.mock.patch` 的 context manager 在多线程场景下存在锁竞争问题
- **修复**：重写 `tests/test_f6_parallel.py`（v3），删除 `patch_openai()` 的导入和调用，改为依赖 `FROST_TESTING=1` 环境变量（进程级设置，对所有线程透明），同时为每个 `thread.join()` 添加超时（60-90s）防止无限等待
- **验证**：PAR 4/4 通过
