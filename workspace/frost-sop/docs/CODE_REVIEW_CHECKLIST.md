# FROST-SOP 代码审查清单

> **目标**：在代码合并前捕获潜在Bug，防止"测了这么多还有Bug"的问题

**使用方式**：
1. 每次代码审查（Code Review）时，逐项检查
2. 任何一项不通过，不允许合并
3. 本清单随项目演进而更新

---

## ✅ 1. 文件操作安全性

### 1.1 文件读取
- [ ] **空文件检查**：读取前检查文件大小 `os.path.getsize(filepath) > 0`
- [ ] **文件存在检查**：使用 `os.path.exists()` 或 `try/except`
- [ ] **权限检查**：验证有读取权限（特别是Linux/Unix）
- [ ] **编码处理**：明确指定 `encoding="utf-8"`，不依赖系统默认编码
- [ ] **try/except包裹**：所有文件操作都在 `try/except` 中

**反例**（不允许）：
```python
# ❌ 危险：不检查空文件
with open(filepath) as f:
    data = json.load(f)
```

**正例**（必须）：
```python
# ✅ 安全：检查空文件 + 异常处理
if os.path.exists(filepath) and os.path.getsize(filepath) > 0:
    try:
        with open(filepath, encoding="utf-8") as f:
            data = json.load(f)
    except json.JSONDecodeError:
        data = {}
else:
    data = {}
```

### 1.2 文件写入
- [ ] **原子写入**：先写临时文件，再重命名（防止崩溃导致文件损坏）
- [ ] **目录存在**：写入前检查 `os.makedirs(os.path.dirname(path), exist_ok=True)`
- [ ] **权限设置**：写入后设置正确权限

**正例**：
```python
# ✅ 安全：原子写入
temp_path = filepath + ".tmp"
with open(temp_path, "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
os.replace(temp_path, filepath)  # 原子操作
```

## ✅ 2. JSON解析安全性

### 2.1 解析前验证
- [ ] **空字符串检查**：`if content.strip():`
- [ ] **try/except包裹**：捕获 `json.JSONDecodeError`
- [ ] **默认值**：解析失败时返回空字典/空列表

### 2.2 解析后验证
- [ ] **结构验证**：检查必要的key是否存在
- [ ] **类型验证**：验证数据类型是否符合预期

**正例**：
```python
try:
    data = json.loads(content)
    if not isinstance(data, dict):
        data = {}
except json.JSONDecodeError:
    data = {}
```

## ✅ 3. 输入验证

### 3.1 API输入
- [ ] **参数存在性**：检查必需参数是否存在
- [ ] **参数类型**：验证参数类型是否正确
- [ ] **参数范围**：验证数值在合理范围内
- [ ] **字符串长度**：防止超长字符串（DoS攻击）
- [ ] **特殊字符**：过滤/转义特殊字符（防止注入）

### 3.2 文件输入
- [ ] **文件大小限制**：防止超大文件（DoS攻击）
- [ ] **文件类型检查**：验证文件扩展名/MIME类型
- [ ] **路径遍历检查**：防止 `../../../etc/passwd` 攻击

**正例**：
```python
# ✅ 安全：路径检查
if not filepath.startswith(ALLOWED_DIR):
    raise ValueError("路径不允许")
```

## ✅ 4. 错误处理

### 4.1 异常处理
- [ ] **具体异常类型**：不使用裸 `except:`，而是指定异常类型
- [ ] **错误信息**：错误信息包含足够上下文（不是只有"Error"）
- [ ] **错误日志**：记录完整堆栈 `traceback.format_exc()`
- [ ] **用户友好**：向用户返回友好错误消息，不暴露内部细节

### 4.2 外部调用
- [ ] **超时设置**：所有网络调用都有超时
- [ ] **重试逻辑**：失败后有重试机制（指数退避）
- [ ] **降级策略**：外部服务不可用时有降级方案

**正例**：
```python
import requests

try:
    response = requests.get(url, timeout=10)
    response.raise_for_status()
except requests.Timeout:
    # 重试或降级
    return fallback_response()
except requests.RequestException as e:
    logger.error(f"API调用失败: {e}")
    return {"error": "服务暂时不可用"}
```

## ✅ 5. 数据库操作

### 5.1 SQL注入防护
- [ ] **参数化查询**：不使用字符串拼接SQL
- [ ] **白名单验证**：表名/列名使用白名单验证

**反例**（不允许）：
```python
# ❌ 危险：SQL注入
query = f"SELECT * FROM {table_name} WHERE id = '{user_id}'"
```

**正例**（必须）：
```python
# ✅ 安全：参数化查询
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))
```

### 5.2 连接管理
- [ ] **连接关闭**：使用 `with` 语句或 `try/finally`
- [ ] **连接池**：使用连接池，不每次创建新连接
- [ ] **事务管理**：正确使用 `commit()` 和 `rollback()`
