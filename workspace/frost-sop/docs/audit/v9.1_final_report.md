## 阶段1: 核心修复 (4个子阶段)

### 1.1 前端状态同步修复 ✅
**用户反馈**: SOP 作战地图没有下拉、对话按钮点不动、看不到阶段详情

**修复内容**:
- `page.tsx` 语法修复 (Badge 导入错误)
- ScrollArea 可滚动容器 (h-[400px])
- 真实阶段数据从 `getTaskStages` API 获取
- 对话按钮状态控制: pending→禁用(🔒), 其他→启用(💬)
- 任务列表点击选中查看阶段详情

**审计**: 19/19 检查项通过

### 1.2 人类府兵超时机制 ✅
**用户反馈**: 人不一定会及时跟进，需要超时容忍度

**修复内容**:
- DB 迁移: `timeout_minutes`, `expires_at`, `timeout_action` 字段
- `check_expired_decisions()`: 查找已超时 pending 决策点
- `resolve_decision_timeout()`: 处理超时 (escalate/abort/auto_approve)
- API: `POST /api/decisions/check-timeout`

**审计**: 7/7 检查项通过

### 1.3 API 错误处理标准化 ✅
**硬性要求**: 【零静默失败】禁止任何空 except 块

**修复内容**:
- 全局异常处理器: `@app.exception_handler(Exception)`
  - 生成唯一 error_id
  - 结构化错误响应: `{ error_code, error_id, message, path }`
- 清理 7 个空 except 块，全部添加 WARN/ERROR 日志
- `get_task_stages` 添加完整异常处理 (输入校验 → DB 查询 → 资源限制)

**审计**: 0 个空 except 块, 10 个错误码定义

### 1.4 前端字数限制修复 ✅
**用户反馈**: 对话输入有字数限制，长需求被截断

**修复内容**:
- `CeoChat.tsx`: `<Input>` → `<Textarea>` (多行文本域)
- 实时字数统计: `{input.length} 字`
- `Shift+Enter` 换行, `Enter` 发送
- 后端: `_max_tokens`: 1024 → 4096

**审计**: 8/8 前端检查 + 2/2 后端检查通过

## 阶段2: 架构加固 (3个任务)

### 2.1 前端 Error Boundary ✅
- `ErrorBoundary.tsx`: React 错误边界组件
- 结构化错误日志 (错误类型、信息、堆栈、时间)
- 友好错误 UI (重试按钮 + 刷新页面按钮)
- `SmallErrorBoundary`: 局部组件包裹版本
- 集成到 `providers.tsx` 包裹整个应用

### 2.2 SSE 断线重连增强 ✅
- 连接状态监控: `connecting/connected/disconnected/error`
- 最大重连次数限制: 10 次
- 心跳超时检测: 35s 无消息强制重连
- 指数退避: 1s → 2s → 4s → ... → 30s
- `SSEStatusIndicator` 组件

### 2.3 API 响应标准化 ✅
- `APIError` 类: 统一错误格式 (status, errorCode, errorId, path)
- `AbortController` 超时控制: 30s 默认超时
- 结构化错误解析: 后端 `{ error_code, error_id, message }` → 前端 `APIError`
- 网络错误 / 超时错误分类处理
- 前端输入校验: PROJECT_ID_INVALID, TASK_ID_INVALID, CHAT_MESSAGE_EMPTY
