推荐使用上下文截断方案，因为实现简单且效果好。

---

## 方案1：上下文截断

**来源**: Suna项目

**核心思路**: 当对话历史过长时，截断中间的消息，只保留最近的和最重要的。

**实现方法**:
```
1. 保留系统消息
2. 保留最近N条消息
3. 截断中间的消息（工具结果保留首尾）
```

**效果**: Token使用降低50-70%

**优点**:
- 实现简单
- 不需要额外LLM调用
- 稳定可靠

**缺点**:
- 可能丢失中间信息

---

## 方案2：LLM生成摘要

**来源**: Gemini CLI项目

**核心思路**: 用LLM对长对话生成摘要，替换原始历史。

**实现方法**:
```
1. 当对话超过阈值，触发摘要生成
2. 用便宜的LLM生成结构化摘要
3. 摘要替换原始历史
```

**效果**: Token使用降低70-80%

**优点**:
- 信息保留率高
- 适合超长对话

**缺点**:
- 需要额外LLM调用
- 增加成本

---

## 方案3：RAG检索

**来源**: Qwen Agent项目

**核心思路**: 将历史对话存储到Vector DB，按需检索。

**实现方法**:
```
1. 保存历史到文件
2. 用RAG检索相关片段
3. 只注入检索到的内容
```

**效果**: Token使用降低80-90%

**优点**:
- Token消耗最低
- 支持无限长对话

**缺点**:
- 需要部署Vector DB
- 实现复杂
