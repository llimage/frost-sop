**目标**: 实现API Key等敏感数据的加密存储

**设计参考**: `02.5-安全设计.md` 第3.2节

**接口定义**:
```python
class SecureConfig:
    def __init__(self, config_path: str = "data/config.json")
    def get(self, key: str, default=None) -> any           # 获取明文配置
    def get_encrypted(self, key: str) -> Optional[str]     # 获取并解密
    def set(self, key: str, value: any, encrypt: bool = False)  # 设置（可选加密）
    def _get_or_create_key(self) -> bytes                  # 生成/读取Fernet密钥
```

**关键实现点**:
- Fernet对称加密

- 文件权限 0o600（仅所有者可读写）
- SQLite `config` 表仅存储非敏感配置

```python
@dataclass
class LLMResponse:
    content: str
    model: str
    provider: str
    input_tokens: int
    output_tokens: int
    cost: float
    latency_ms: int
    error: Optional[str] = None

class LLMClient:
    def __init__(self, config: SecureConfig, cost_tracker: CostTracker)
    def chat(self, messages: list[dict], model_preference: str = 'auto',
             temperature: float = 0.7, max_tokens: Optional[int] = None,
             agent_id: str = 'unknown') -> LLMResponse

    # 内部方法
    def _decide_model(self, preference: str, messages: list[dict]) -> bool
```

### 3.3 PermissionChecker (权限检查)

**目标**: 实现P0-P4五级权限模型

**设计参考**: `02.5-安全设计.md` 第2节

**权限级别**:
- P0: 创始人专用（系统关机、配置修改、数据删除）
- P1: 读取权限（文件读取、数据库查询、日志查看、LLM调用）
- P2: 写入权限（文件写入、数据库修改、Git提交）
- P3: 执行权限（代码执行、命令执行）
- P4: 管理权限（Agent启停、配置变更、权限分配）

**Agent默认权限**:
- 主Agent: P1, P2
- 审计Agent: P1
- 开发Agent: P1, P2, P3

**接口定义**:
```python
@dataclass
class PermissionResult:
    allowed: bool
    reason: str
    level: Optional[PermissionLevel]

class PermissionChecker:
    def __init__(self, db: Database)
    def check_permission(self, subject: str, operation: str,
                         target: Optional[str] = None) -> PermissionResult
    def _get_agent_permissions(self, agent_id: str) -> list[PermissionLevel]
    def _check_target_restrictions(self, agent_id: str, operation: str, target: str) -> bool
```

**禁止路径**:
- `.env`
- `data/solo_ops.db`
- `data/chromadb`
- `.git/config`
- `__pycache__`
- 项目目录外的路径

```python
class AgentResult:
```

### 4.2 MainAgent (主Agent)

**目标**: 意图解析、SOP选择、团队组建、任务协调

**设计参考**: `02.1-系统架构设计.md` 第3.2节

**核心功能**:
1. **意图解析**: 调用本地3B模型识别用户指令类型
2. **SOP匹配**: 根据意图选择 DEV-001 / DEV-002 / STR-002
3. **阶段规划**: 分解SOP为执行阶段，确定串行/并行
4. **团队组建**: 决定需要哪些Agent参与
5. **结果汇总**: 收集各Agent输出，整合为最终交付物

**接口定义**:
```python
class MainAgent(BaseAgent):
    def parse_intent(self, user_input: str) -> dict  # {intent, confidence, sop_id}
    def select_sop(self, intent: dict) -> Optional[dict]  # SOP模板
    def plan_stages(self, sop: dict, context: dict) -> list[dict]  # 阶段列表
    def execute_stage(self, stage: dict, agents: list) -> dict  # 阶段结果
    def compile_result(self, task_id: str) -> str  # 汇总最终结果
```

**意图类型**:
- `dev` - 软件开发（匹配DEV-001）
- `bugfix` - Bug修复（匹配DEV-002）
- `review` - 周期性回顾（匹配STR-002）
- `chat` - 闲聊/问答（无SOP，直接回答）
- `unknown` - 无法识别（询问用户）

**测试要点**:
- [ ] 意图解析准确率 >60%
- [ ] SOP匹配正确
- [ ] 阶段规划完整
- [ ] 结果汇总合理

### 4.3 AuditAgent (审计Agent)

**目标**: 代码审查、成本监督、质量检查、技术建议

**设计参考**: `02.1-系统架构设计.md` 第3.3节

**核心功能**:
1. **代码审查**: 检查代码规范、安全性、潜在bug
2. **成本监督**: 监控Token消耗，接近阈值时预警
3. **质量检查**: 检查SOP执行结果是否符合验收标准
4. **技术建议**: 提供建议，**不裁决**（最终决策权归创始人）

**权限边界**:
- ✅ 可以: 审查代码、记录问题、提供建议、预警成本
- ❌ 不可以: 自动修改代码、自动拒绝任务、强制停止执行

**接口定义**:
```python
class AuditAgent(BaseAgent):
    def review_code(self, code: str, language: str = "python") -> dict
        # 返回: {pass, issues: [{severity, message, line}], suggestions}

    def check_cost(self, current_month_cost: float) -> dict
        # 返回: {status: normal/warning/cutoff, message}

    def review_quality(self, deliverable: str, criteria: list) -> dict
        # 返回: {score, pass, feedback}

    def advise_technical(self, proposal: str) -> dict
        # 返回: {advice, risks, alternatives}
```

**代码审查标准**:
- 致命: SQL注入、路径遍历、命令注入
- 警告: 未关闭资源、异常处理缺失、硬编码
- 建议: 代码风格、注释、优化建议

**测试要点**:
- [ ] 代码审查发现SQL注入
- [ ] 成本预警在80%时触发
- [ ] 质量评分合理
- [ ] 不自动修改代码

### 4.5 Scheduler (调度器)

**目标**: 系统中枢，连接前端事件与Agent执行，管理任务队列和状态广播

**设计参考**: `02.1-系统架构设计.md` 第3.1节

**状态机**:
```
IDLE → PARSING → RUNNING → WAITING → RUNNING → COMPLETED
              ↓         ↓
           FAILED   CANCELLED
```

**接口定义**:
```python
class Scheduler:
    def __init__(self, db: Database, memory: MemoryStore,
                 config: SecureConfig, llm: LLMClient)

    # 用户交互
    def on_user_input(self, text: str) -> str  # 任务ID
    def on_user_confirm(self, task_id: str, decision: str) -> None
    def on_user_cancel(self, task_id: str) -> None

    # Agent管理
    def get_agent_status(self) -> dict  # {agent_id: {status, progress, current_task}}
    def get_task_result(self, task_id: str) -> dict
    def get_today_tasks(self) -> list[dict]

    # 消息总线
    def send_message(self, msg: AgentMessage) -> bool
    def receive_message(self, agent_id: str, timeout: float = 0.1) -> Optional[AgentMessage]

    # 内部
    def _dispatch_task(self, task_id: str) -> None  # 后台线程
    def _update_ui(self) -> None  # 定时刷新前端
```

**消息总线**:
- 内存Queue（Python标准库）
- 调度器中转（MVP不直接点对点）
- 自动记录audit_log

**测试要点**:
- [ ] 用户输入创建任务成功
- [ ] Agent状态更新正确
- [ ] 消息发送/接收正常
- [ ] 决策点暂停/恢复正常
- [ ] 任务取消正常
