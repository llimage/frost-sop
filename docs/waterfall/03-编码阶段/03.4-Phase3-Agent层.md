## 3.1 BaseAgent (`agents/base.py`)

**功能**: 所有Agent的抽象基类，定义公共接口和状态管理

**设计参考**: `docs/waterfall/02-系统设计/02.1-系统架构设计.md` 第5.1节

### 接口定义

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional, Dict, Any, List
from datetime import datetime

from core.database import Database
from core.memory import MemoryStore
from core.toolkit import ToolKit
from core.llm_client import LLMClient, LLMResponse


@dataclass
class AgentConfig:
    """Agent配置"""
    agent_id: str
    name: str
    role: str
    model_preference: str           # "local" / "cloud" / "auto"
    memory_collection: str          # ChromaDB collection名称
    available_tools: List[str]    # 可用工具列表
    system_prompt: str              # 系统提示词
    color: str = "#4F46E5"         # UI显示颜色
    icon: str = "🤖"               # UI显示图标


@dataclass
class AgentResult:
    """Agent执行结果"""
    success: bool
    output: str
    cost: float
    tokens: int
    latency_ms: int
    error: Optional[str] = None


class BaseAgent(ABC):
    """Agent抽象基类
    
    所有常驻Agent和角色模板的基类。
    提供: 状态管理、记忆操作、LLM调用、工具调用
    """
    
    def __init__(self,
                 config: AgentConfig,
                 db: Database,
                 memory: MemoryStore,
                 tools: ToolKit,
                 llm: LLMClient):
        """初始化Agent
        
        流程:
        1. 保存config, db, memory, tools, llm
        2. 设置初始状态: idle
        3. 当前任务: None
        """
        self.config = config
        self.db = db
        self.memory = memory
        self.tools = tools
        self.llm = llm
        self.status = "idle"           # idle/running/waiting/paused/error
        self.current_task = None
    
    @abstractmethod
    def execute(self, task: Dict[str, Any]) -> AgentResult:
        """执行分配的任务 (子类必须实现)
        
        Args:
            task: {
                "task_id": str,
                "type": str,           # 任务类型
                "description": str,      # 任务描述
                "context": str,          # 上下文 (前序输出)
                "params": dict,          # 额外参数
            }
        
        Returns:
            AgentResult
        """
        pass
    
    def update_status(self, status: str, progress: Optional[int] = None) -> None:
        """更新Agent状态并广播到前端
        
        流程:
        1. 更新self.status
        2. 写入agent_status表 (INSERT新记录)
        3. 更新agents表的status字段
        
        状态值: idle/running/waiting/paused/error/monitoring
        progress: 0-100 (可选)
        """
        pass
    
    def recall_memory(self, query: str, top_k: int = 3) -> str:
        """检索相关记忆，格式化为上下文
        
        流程:
        1. 调用memory.search_memory(agent_id, query, top_k)
        2. 格式化结果为文本
        3. 返回格式: "## 相关记忆\\n1. [记忆内容]\\n2. [...]"
        """
        pass
    
    def save_memory(self, content: str, metadata: Dict[str, Any]) -> None:
        """保存执行结果到记忆库
        
        流程:
        1. 补充metadata: agent_id, timestamp, doc_type
        2. 调用memory.add_memory(agent_id, content, metadata)
        """
        pass
    
    def call_llm(self,
                 messages: List[Dict[str, str]],
                 model_preference: Optional[str] = None,
                 **kwargs) -> LLMResponse:
        """调用LLM (包装llm_client.chat)
        
        自动传入:
        - agent_id = self.config.agent_id
        - model_preference = self.config.model_preference (如未指定)
        """
        pass
    
    def call_tool(self, tool_name: str, params: Dict[str, Any], 
                  task_id: Optional[str] = None) -> Dict:
        """调用工具 (包装toolkit.call_tool)
        
        自动传入:
        - agent_id = self.config.agent_id
        """
        pass
```

### 关键实现点

1. **状态持久化**: 每次update_status同时写入agent_status表(历史)和agents表(当前)
2. **记忆自动关联**: save_memory自动补充agent_id和timestamp到metadata
3. **LLM偏好继承**: call_llm默认使用config.model_preference
4. **工具调用审计**: call_tool自动记录到audit_log

        3. 返回格式化文本

1. **意图解析Prompt**: 必须要求LLM返回严格JSON格式
2. **置信度阈值**: <0.6标记为unknown，询问用户澄清

### 关键实现点

1. **线程安全**: 使用threading.Lock保护共享状态
2. **任务队列**: 使用queue.PriorityQueue (P0最高优先级)
3. **消息总线**: 每个Agent一个Queue，调度器中转
4. **决策点**: 暂停任务线程，等待用户确认 (不阻塞UI)
5. **状态广播**: 后台线程每3秒查询数据库，更新前端绑定变量
6. **错误恢复**: 阶段失败重试1次，仍失败则标记任务failed
