## 2.1 LLMClient (`core/llm_client.py`)

**功能**: 统一LLM调用接口，封装Ollama本地模型和云端API，实现任务类型路由

**设计参考**: `docs/waterfall/02-系统设计/02.3-接口设计.md` 第4节

### 接口定义

```python
import time
import requests
from dataclasses import dataclass
from typing import Optional, List, Dict, Literal

from core.config import SecureConfig
from core.cost_tracker import CostTracker


@dataclass
class LLMResponse:
    """LLM调用统一响应"""
    content: str                    # 生成的内容
    model: str                      # 实际使用的模型
    provider: str                   # 提供商: ollama/deepseek/openai/kimi
    input_tokens: int               # 输入Token数
    output_tokens: int              # 输出Token数
    cost: float                     # 成本(元)
    latency_ms: int                 # 延迟(毫秒)
    error: Optional[str] = None     # 错误信息(如有)


class LLMClient:
    """统一LLM调用客户端

    特性:
    - 本地Ollama优先
    - 任务类型自动路由 (local/cloud/auto)
    - 成本熔断检查
    - Token计数和成本计算
    """

    def __init__(self, config: SecureConfig, cost_tracker: CostTracker):
        """初始化LLMClient

        从config读取:
        - ollama_host (默认 http://localhost:11434)
        - ollama_model (默认 qwen2.5:3b)
        - api_provider (默认 deepseek)
        - api_base_url (默认 https://api.deepseek.com)
        - api_model (默认 deepseek-chat)

        从SecureConfig读取(加密):
        - api_key

        定价表 (元/千Token):
        - ollama: input=0, output=0 (本地0成本)
        - deepseek: input=0.001, output=0.002 (示例，需按实际更新)
        - openai: input=0.003, output=0.006
        - kimi: input=0.002, output=0.004
        """
        pass

    def chat(self,
             messages: List[Dict[str, str]],
             model_preference: Literal['local', 'cloud', 'auto'] = 'auto',
             temperature: float = 0.7,
             max_tokens: Optional[int] = None,
             agent_id: str = 'unknown') -> LLMResponse:
        """统一聊天接口

        流程:
        1. 记录开始时间
        2. 决策使用本地还是云端 (_decide_model)
        3. 调用对应接口 (_call_ollama 或 _call_cloud_api)
        4. 计算延迟
        5. 记录成本 (cost_tracker.record_call)
        6. 返回LLMResponse

        Args:
            messages: OpenAI格式消息列表 [{"role": "system"/"user"/"assistant", "content": "..."}]
            model_preference: 'local'=强制本地, 'cloud'=强制云端, 'auto'=自动选择
            temperature: 温度参数 (0-2)
            max_tokens: 最大输出token数 (默认2048)
            agent_id: 调用者Agent ID (用于成本追踪)
        """
        pass

    def embed(self, texts: List[str]) -> List[List[float]]:
        """调用Embedding模型 (nomic-embed-text via Ollama)

        API: POST {ollama_host}/api/embed
        Body: {"model": "nomic-embed-text", "input": texts}

        返回: embeddings列表，失败返回零向量
        """
        pass

    def _decide_model(self,
                      preference: Literal['local', 'cloud', 'auto'],
                      messages: List[Dict[str, str]]) -> bool:
        """决策使用本地(True)还是云端(False)

        决策流程:
        1. 显式偏好优先: local->True, cloud->False (但检查熔断)
        2. 成本熔断检查: 预算耗尽->强制True (最高优先级)
        3. 任务类型识别: 从messages判断任务类型
        4. 消息长度辅助: 短文本(<1000字)即使复杂任务也尝试本地

        任务类型路由矩阵:
        | 任务类型      | 默认路由 | 理由                  |
        |--------------|---------|----------------------|
        | intent_parse | local   | 短文本分类，3B足够    |
        | code_gen     | cloud   | 代码质量要求高        |
        | code_review  | cloud   | 需发现潜在bug         |
        | summary      | local   | 文本压缩，3B可胜任    |
        | qa           | auto    | 视问题复杂度动态决定  |
        | creative     | cloud   | 创意质量要求高        |
        | data_analysis| local   | 本地有Python辅助      |
        | chat         | local   | 通用对话，3B足够      |

        auto动态决策:
        - 短文本(<1000字) -> local
        - 中等文本(<3000字) + 非代码类 -> local
        - 长文本或代码类 -> cloud
        """
        pass

    def _detect_task_type(self, messages: List[Dict[str, str]]) -> str:
        """从消息中识别任务类型

        识别策略 (按优先级排序):
        1. 代码生成类: 关键词 '生成代码','写代码','code','function','def ','class ','实现','开发功能'
           - 子类: 含'审查','review','检查','audit','bug' -> code_review
           - 否则 -> code_gen
        2. 意图解析类: '意图','intent','分类','识别','解析','判断','属于哪类'
        3. 摘要类: '摘要','总结','summarize','summary','概括','提炼'
        4. 数据分析类: '分析数据','数据分析','统计','计算','pandas','csv','excel'
        5. 创意类: '创意','写作','写文章','文案','营销','creative'
        6. 问答类: '为什么','怎么','如何','什么是','解释','说明','help','question','?'
        7. 默认: chat

        返回: intent_parse/code_gen/code_review/summary/qa/creative/data_analysis/chat
        """
        pass

    def _call_ollama(self,
                     messages: List[Dict[str, str]],
                     temperature: float,
                     max_tokens: Optional[int]) -> LLMResponse:
        """调用本地Ollama

        API: POST {ollama_host}/v1/chat/completions (OpenAI兼容格式)
        Body: {
            "model": ollama_model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens or 2048,
            "stream": false
        }

        超时: 120秒
        成本: 0元

        返回LLMResponse:
        - content: choices[0].message.content
        - model: ollama_model
        - provider: "ollama"
        - input_tokens: usage.prompt_tokens
        - output_tokens: usage.completion_tokens
        - cost: 0.0
        """
        pass

    def _call_cloud_api(self,
                        messages: List[Dict[str, str]],
                        temperature: float,
                        max_tokens: Optional[int]) -> LLMResponse:
        """调用云端API

        API: POST {api_base}/v1/chat/completions
        Headers: Authorization: Bearer {api_key}
        Body: 同上

        超时: (连接5秒, 读取60秒)
        成本: 按定价表计算

        返回LLMResponse:
        - cost: (input_tokens * input_price + output_tokens * output_price) / 1000
        """
        pass
```

### 关键实现点

1. **Ollama API格式**: 使用 `/v1/chat/completions` (OpenAI兼容)，不是 `/api/chat`
2. **Token计数**: Ollama返回 `usage.prompt_tokens` 和 `usage.completion_tokens`
3. **成本计算**: 云端按定价表计算，本地始终0元
4. **熔断检查**: 调用 `_call_cloud_api` 前必须检查 `cost_tracker.check_budget()`
5. **错误处理**: 任何异常返回 `LLMResponse(error=...)`，不抛异常
6. **Embedding**: 使用 `/api/embed` 端点，返回768维向量

1. **月度计算**: 使用SQL `strftime('%Y-%m', timestamp) = strftime('%Y-%m', 'now')`

## 2.3 PermissionChecker (`core/permission.py`)

**功能**: P0-P4五级权限模型，控制Agent可操作范围

**设计参考**: `docs/waterfall/02-系统设计/02.5-安全设计.md` 第2节

### 接口定义

```python
from enum import Enum
from dataclasses import dataclass
from typing import Optional, List, Dict

from core.database import Database


class PermissionLevel(Enum):
    """权限级别"""
    P0 = "founder"      # 创始人: 所有权限
    P1 = "read"         # 读取: 文件读取、数据库查询、日志查看、LLM调用
    P2 = "write"        # 写入: 文件写入、数据库修改、Git提交
    P3 = "execute"      # 执行: 代码执行、命令执行
    P4 = "admin"        # 管理: 配置修改、Agent启停、数据删除


@dataclass
class PermissionResult:
    """权限检查结果"""
    allowed: bool
    reason: str
    level: Optional[PermissionLevel]


class PermissionChecker:
    """权限检查器

    权限矩阵 (操作 -> 所需权限):
    - file_read: P1
    - file_write: P2
    - file_delete: P4
    - db_query: P1
    - db_insert: P2
    - db_update: P2
    - db_delete: P4
    - code_execute: P3
    - code_execute_unrestricted: P0
    - git_status: P1
    - git_commit: P2
    - git_push: P4
    - system_shutdown: P0
    - system_restart: P0
    - config_read: P1
    - config_write: P4
    - agent_start: P4
    - agent_stop: P4
    - agent_config: P4

    Agent默认权限:
    - agent_main: [P1, P2]
    - agent_audit: [P1]
    - agent_dev: [P1, P2, P3]

    禁止路径:
    - .env
    - data/solo_ops.db
    - data/chromadb
    - .git/config
    - __pycache__
    - 项目目录外的路径
    """

    def __init__(self, db: Database):
        """初始化PermissionChecker

        从数据库加载Agent权限配置
        """
        pass

    def check_permission(self,
                         subject: str,           # "user" 或 agent_id
                         operation: str,
                         target: Optional[str] = None) -> PermissionResult:
        """检查主体是否有权限执行操作

        流程:
        1. 创始人(user) -> 直接通过 (P0)
        2. 获取Agent权限列表
        3. 获取操作所需权限
        4. 检查权限是否满足
        5. 检查目标对象特殊限制 (禁止路径)
        6. 记录权限检查日志

        返回PermissionResult:
        - allowed: True/False
        - reason: 说明
        - level: 所需权限级别
        """
        pass

    def _get_agent_permissions(self, agent_id: str) -> List[PermissionLevel]:
        """获取Agent权限列表

        从数据库agents表读取permissions字段
        格式: "P1,P2" 字符串，解析为列表
        不存在则返回默认权限
        """
        pass

    def _check_target_restrictions(self,
                                    agent_id: str,
                                    operation: str,
                                    target: str) -> bool:
        """检查目标对象特殊限制

        检查项:
        1. 目标是否在禁止路径列表中
        2. 目标路径是否在项目目录内 (防止目录遍历)

        返回: True=允许, False=禁止
        """
        pass
```

### 关键实现点

1. **创始人特权**: `subject == "user"` 时直接返回allowed=True
2. **禁止路径检查**: 使用字符串包含检查 + 绝对路径前缀检查
3. **权限存储**: agents表的permissions字段存储逗号分隔的权限级别
4. **日志记录**: 每次权限检查记录到audit_log

```python
class CodeSandbox:
    """代码执行沙箱 (Windows兼容)

    安全措施:
    1. 语法检查 (compile())
    2. 禁止模式检查 (subprocess, os.system, eval, exec, __import__)
    3. 子进程隔离 (subprocess.run)
    4. 超时控制 (30秒)
    5. 工作目录限制 (cwd=os.getcwd())

    注意: Windows不支持Unix的resource模块进行内存限制
    安全边界依赖: 语法检查 + 禁止模式 + subprocess超时 + cwd限制
    """

    def __init__(self, timeout: int = 30, max_memory_mb: int = 512):
        self.timeout = timeout
        self.max_memory_mb = max_memory_mb
        self.allowed_modules = [
            'os', 'sys', 'json', 're', 'datetime', 'math', 'random',
            'collections', 'itertools', 'functools', 'typing',
            'sqlite3', 'pathlib', 'hashlib', 'base64', 'urllib.parse',
            'requests',
        ]
        self.forbidden_patterns = [
            'import subprocess',
            'os.system',
            'os.popen',
            'eval(',
            'exec(',
            '__import__',
            'open("/etc',
            'open("C:\\Windows',
        ]

    def execute(self, code: str, context: Optional[Dict] = None) -> Dict:
        """在沙箱中执行代码

        流程:
        1. 语法检查
        2. 禁止模式检查
        3. 创建临时文件 (写入安全包装代码)
        4. subprocess.run执行 (带超时)
        5. 清理临时文件

        返回: {
            "success": bool,
            "stdout": str,
            "stderr": str,
            "error": str (如有)
        }
        """
        pass
```

### 关键实现点

1. **权限检查**: 调用tool前必须通过PermissionChecker
2. **审计日志**: 每次tool调用记录到audit_log (operation_type='tool_call')
3. **CodeSandbox**:
   - 使用 `tempfile.NamedTemporaryFile` 创建临时py文件
   - 包装代码设置 `os.chdir(os.getcwd())` 限制工作目录
   - `subprocess.run(['python', temp_path], timeout=30, cwd=os.getcwd())`
   - 禁止模式用字符串包含检查 (简单但有效)
4. **错误处理**: 权限拒绝返回 `{"success": False, "error": "权限拒绝: ..."}`
