### SOP模板格式

```yaml
sop_id: "DEV-001"
name: "新功能开发"
category: "dev"          # dev/mkt/ops/str
version: "1.0"
description: "标准软件开发流程，包含需求→设计→实现→审查→测试"
priority: "P1"           # P0/P1/P2/P3/P4
total_token_estimate: 2200
stages:
  - stage_num: 1
    name: "需求讨论与确认"
    mode: "parallel"      # parallel / serial
    agents: ["agent_pm", "agent_arch"]  # 角色模板名称
    input: "原始需求描述"
    output: "PRD文档 + 技术方案"
    decision_point: true
    decision_options: ["确认", "驳回"]
    token_estimate: 800
    
  - stage_num: 2
    name: "设计评审"
    mode: "serial"
    agents: ["agent_arch", "agent_dev"]
    input: "阶段1输出物"
    output: "详细设计 + 接口定义"
    decision_point: true
    decision_options: ["确认", "驳回"]
    token_estimate: 600
    
  - stage_num: 3
    name: "开发实现"
    mode: "serial"
    agents: ["agent_dev"]
    input: "阶段2输出物"
    output: "代码 + 单元测试"
    decision_point: false
    token_estimate: 1200
    
  - stage_num: 4
    name: "测试验证"
    mode: "serial"
    agents: ["agent_test"]
    input: "阶段3输出物"
    output: "测试报告"
    decision_point: false
    token_estimate: 400
    
  - stage_num: 5
    name: "审计封板"
    mode: "serial"
    agents: ["agent_audit"]
    input: "阶段4输出物"
    output: "审计报告"
    decision_point: true
    decision_options: ["确认", "驳回"]
    token_estimate: 200
```

## 4.2 SOPEngine (`core/sop_engine.py`)

**功能**: SOP模板加载、阶段执行、状态流转、决策点管理

**设计参考**: `docs/waterfall/02-系统设计/02.1-系统架构设计.md` 第3.5节

### 接口定义

```python

import json
from pathlib import Path
from typing import Dict, List, Optional, Any
from datetime import datetime

from core.database import Database


class SOPEngine:
    """SOP执行引擎
    
    职责:
    1. 从磁盘加载SOP模板 (YAML)
    2. 将模板同步到数据库 (sop_templates表)
    3. 执行SOP (按阶段顺序)
    4. 管理阶段状态流转
    5. 处理决策点 (暂停等待用户)
    6. 阶段失败重试1次

    def __init__(self, sop_dir: str = "sops", db: Optional[Database] = None):

        1. 保存sop_dir和db
        2. 加载所有SOP模板 (_load_all_sops)
        3. 如db存在，同步到数据库
        """
        self.sop_dir = Path(sop_dir)
        self.db = db

        self._load_all_sops()
    
    def _load_all_sops(self) -> None:
        """加载所有SOP模板
        
        流程:
        1. 遍历sop_dir下所有.yaml文件
        2. 解析YAML为dict
        3. 验证格式 (必须有sop_id, name, stages)
        4. 存入self.sops
        5. 如db存在，插入/更新sop_templates表
        """

    def load_sop(self, sop_id: str) -> Optional[Dict[str, Any]]:
        """加载指定SOP模板
        
        优先从内存缓存读取，不存在则从数据库读取
        
        Returns:
            SOP模板dict 或 None
        """
        pass
    
    def list_sops(self, category: Optional[str] = None) -> List[Dict[str, Any]]:
        """列出所有可用SOP
        
        Args:
            category: 可选过滤 (dev/mkt/ops/str)
        
        Returns:
            SOP模板列表
        """
        pass
    
    def execute_sop(self,
                    sop_id: str,
                    task_id: str,
                    context: Dict[str, Any],
                    scheduler: Any) -> Dict[str, Any]:
        """执行完整SOP
        
        流程:
        1. 加载SOP模板
        2. 创建sop_executions记录 (status=running)
        3. 遍历所有阶段:
           a. 更新current_stage
           b. 调用execute_stage()
           c. 如decision_point=true -> 暂停，返回等待状态
           d. 如阶段失败 -> 重试1次，仍失败则标记SOP失败
        4. 所有阶段完成 -> 标记SOP完成
        5. 返回执行结果
        
        Args:
            sop_id: SOP模板ID
            task_id: 关联任务ID
            context: 执行上下文 (用户输入、参数等)
            scheduler: 调度器实例 (用于Agent通信和决策点)
        
        Returns:
            {
                "success": bool,
                "task_id": str,
                "sop_id": str,
                "stages_completed": int,
                "stages_total": int,
                "status": "completed"|"failed"|"cancelled"|"waiting",
                "waiting_stage": Optional[int],  # 如等待用户确认
                "results": [stage_result],
                "error": Optional[str]
            }
        """
        pass
    
    def execute_stage(self,
                      stage: Dict[str, Any],
                      task_id: str,
                      context: Dict[str, Any],
                      scheduler: Any) -> Dict[str, Any]:
        """执行单个阶段

        4. 通过scheduler分发任务给Agent
        5. 等待Agent完成 (通过消息总线)
        6. 保存阶段结果到task_stages

        Args:

            task_id: 任务ID

            scheduler: 调度器实例
        
        Returns:
            {
                "stage_id": str,
                "status": "completed"|"failed",
                "output": str,
                "cost": float,
                "tokens": int,
                "error": Optional[str]
            }
        """
        pass
    
    def pause_at_decision(self,
                          task_id: str,
                          stage_num: int,
                          question: str,
                          options: List[str],
                          scheduler: Any) -> None:
        """在决策点暂停
        
        流程:
        1. 更新task_stages状态为waiting
        2. 更新sop_executions状态为waiting
        3. 通过scheduler发送decision消息到前端
        4. 记录audit_log
        
        暂停后，等待用户通过scheduler.on_user_confirm()恢复
        """
        pass
    
    def resume_after_decision(self,

                              stage_num: int,

        3. 如decision="驳回" -> 标记阶段失败，可能需要重试

                "stages": [

                        "stage_num": int,

                        "status": str,
```

### 关键实现点

1. **YAML加载**: 使用 `yaml.safe_load()` 解析
2. **数据库同步**: 启动时自动将YAML同步到sop_templates表
3. **阶段执行**: 通过scheduler分发任务，不直接调用Agent
4. **决策点**: 暂停SOP执行线程，等待用户确认 (不阻塞UI)
5. **失败重试**: 阶段失败自动重试1次，仍失败则SOP失败
6. **进度追踪**: 实时更新sop_executions和task_stages表

### 阶段状态流转

```
pending -> running -> completed
              ↓
           failed (重试1次) -> failed (最终)
              ↓
           waiting (决策点) -> confirmed -> running
                          -> rejected -> failed
```
