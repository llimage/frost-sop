## 1.1 SecureConfig (`core/config.py`)

**功能**: API Key等敏感数据的Fernet加密存储

**设计参考**: `docs/waterfall/02-系统设计/02.5-安全设计.md` 第3.2节

### 接口定义

```python
from cryptography.fernet import Fernet
import os
import json
import base64
from pathlib import Path
from typing import Optional, Any


class SecureConfig:
    """安全配置管理器 —— 敏感数据唯一存储位置
    
    设计原则:
    - SQLite config表仅存储非敏感配置（预算、阈值、主题等）
    - 所有敏感数据（API Key、密码等）必须通过此类存储
    - 使用Fernet对称加密，密钥存储在data/.secret_key
    """
    
    def __init__(self, config_path: str = "data/config.json"):
        """初始化SecureConfig
        
        流程:
        1. 确保data目录存在
        2. 获取或创建Fernet密钥 (data/.secret_key)
        3. 初始化cipher
        4. 加载现有配置
        """
        pass
    
    def _get_or_create_key(self) -> bytes:
        """获取或创建Fernet加密密钥
        
        流程:
        1. 检查 data/.secret_key 是否存在
        2. 存在: 读取文件，base64解码返回
        3. 不存在: 生成新密钥，base64编码写入文件，设置权限0o600
        """
        pass
    
    def _load_config(self) -> dict:
        """加载配置文件
        
        流程:
        1. 检查 config_path 是否存在
        2. 存在: JSON加载返回
        3. 不存在: 返回空dict
        """
        pass
    
    def _save_config(self) -> None:
        """保存配置到文件
        
        流程:
        1. 确保目录存在
        2. JSON写入config_path (indent=2, ensure_ascii=False)
        3. 设置文件权限0o600
        """
        pass
    
    def get(self, key: str, default: Any = None) -> Any:
        """获取明文配置项 (非敏感数据)"""
        pass
    
    def get_encrypted(self, key: str) -> Optional[str]:
        """获取加密配置项并解密 (敏感数据如API Key)
        
        流程:
        1. 从_config获取加密值
        2. 使用cipher.decrypt解密
        3. 返回解密后的字符串
        4. 失败返回None
        """
        pass
    
    def set(self, key: str, value: Any, encrypt: bool = False) -> None:
        """设置配置项
        
        Args:
            key: 配置键
            value: 配置值
            encrypt: True=加密存储(敏感数据), False=明文存储
        """
        pass
```

### 关键实现点

1. **密钥管理**: `data/.secret_key` 存储base64编码的Fernet密钥
2. **文件权限**: Windows用 `os.chmod(path, 0o600)`，可能不生效但保留代码
3. **配置分离**: 明文配置存 `config.json`，加密配置也存同一文件但值是密文
4. **首次启动**: 自动生成密钥和空配置

## 1.2 Database (`core/database.py`)

### 接口定义

```python
        pass

        """事务上下文管理器
        
        Usage:
            with db.transaction():
                db.execute("INSERT ...")
                db.execute("UPDATE ...")
        """
        pass
    
    def init_schema(self) -> None:
        """初始化数据库Schema
        
        创建以下表:

        pass
    
    def backup(self, backup_path: Optional[str] = None) -> str:
        """创建数据库备份
        
        使用SQLite内置backup API
        返回备份文件路径
        """
        pass
```

### 关键实现点

1. **连接管理**: 单连接复用，每次操作后commit
2. **参数化查询**: 所有execute/query必须传params列表，禁止字符串拼接SQL
3. **安全检查**: 
   - `if "DROP TABLE" in sql.upper(): raise SecurityError`
   - `if "DELETE FROM audit_log" in sql.upper(): raise SecurityError`
4. **触发器**: 
   ```sql
   CREATE TRIGGER prevent_audit_log_delete BEFORE DELETE ON audit_log
   BEGIN SELECT RAISE(ABORT, '审计日志禁止删除'); END;
   
   CREATE TRIGGER prevent_audit_log_update BEFORE UPDATE ON audit_log
   BEGIN SELECT RAISE(ABORT, '审计日志禁止修改'); END;
   ```
5. **初始数据**: 严格按设计文档02.2中的INSERT语句

## 1.3 MemoryStore (`core/memory.py`)

**设计参考**: `docs/waterfall/02-系统设计/02.2-数据库设计.md` 第3节

### 接口定义

### 关键实现点

1. **ChromaDB配置**: 
   ```python
   Settings(anonymized_telemetry=False, allow_reset=True)
   ```
2. **Collection元数据**: `{"hnsw:space": "cosine"}`
3. **Embedding API**: Ollama的 `/api/embed` 端点 (不是 `/v1/embeddings`)
4. **容错**: Ollama未启动时返回零向量，不阻塞系统
5. **文档ID**: 使用 `metadata.get("doc_id", f"mem_{uuid}_{timestamp}")`
