## 1. 设计目标与约束

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| G1 | 单文件零配置：SQLite 单文件 `.db`，ChromaDB 单目录，无需服务器 |
| G2 | 数据量支撑 3 年运营：任务 10,000+、日志 100,000+、记忆 5,000+ |
| G3 | 查询性能：关键查询 <100ms（任务列表、Agent 状态、成本汇总） |
| G4 | 数据完整性：外键约束、事务保证、审计日志不可篡改 |
| G5 | 可导出：SQLite 可直接复制，ChromaDB 可导出 JSON |

### 1.2 设计约束

| 约束 | 来源 | 影响 |
|------|------|------|
| 单用户系统 | PRD | 无需用户表、权限表、会话管理 |
| 本地部署 | PRD | 无需网络优化、分片、集群 |
| 内存 ≤3.5GB | PRD | SQLite 缓存大小限制、ChromaDB 索引策略 |
| 创始人 IT 小白 | PRD | Schema 变更需自动迁移，不能手动执行 SQL |
| 追加-only 审计日志 | PRD | 审计表无 UPDATE/DELETE，只有 INSERT |

### 1.3 数据规模预估

| 数据类型 | 预估 3 年总量 | 单条大小 | 总大小 |
|---------|-------------|---------|--------|
| 任务记录 | 10,000 | 2KB | 20MB |
| 审计日志 | 100,000 | 1KB | 100MB |
| 成本记录 | 50,000 | 0.5KB | 25MB |
| 能量记录 | 1,000 | 0.5KB | 0.5MB |
| 日程记录 | 2,000 | 1KB | 2MB |
| Agent 记忆（文本） | 5,000 | 2KB | 10MB |
| **SQLite 总计** | — | — | **~160MB** |
| ChromaDB 向量 | 5,000 | 4KB (768d×4byte) | 20MB |
| ChromaDB 元数据 | 5,000 | 1KB | 5MB |
| **ChromaDB 总计** | — | — | **~25MB** |
| 输出文件 | 3,000 | 10KB | 30MB |
| **总数据量** | — | — | **~215MB** |

**结论**：数据量极小，SQLite + ChromaDB 完全够用，无需优化存储空间。

#### `tasks` —— 任务记录

```sql
CREATE TABLE tasks (
    task_id         TEXT PRIMARY KEY,           -- 任务ID: task_{timestamp}_{shortid}
    user_input      TEXT NOT NULL,              -- 创始人原始输入
    intent          TEXT,                       -- 解析后的意图类型
    intent_confidence REAL,                     -- 意图置信度(0-1)
    sop_id          TEXT,                       -- 匹配的SOP模板ID
    priority        TEXT NOT NULL DEFAULT 'P2', -- P0/P1/P2/P3/P4
    status          TEXT NOT NULL DEFAULT 'pending', -- pending/running/paused/completed/failed/cancelled
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    started_at      TIMESTAMP,                  -- 开始执行时间
    completed_at    TIMESTAMP,                  -- 完成时间
    result_summary  TEXT,                       -- 结果摘要
    result_path     TEXT,                       -- 结果文件路径
    cost_total      REAL DEFAULT 0.0,           -- 任务总成本(元)
    token_total     INTEGER DEFAULT 0,          -- 任务总Token数
    error_message   TEXT,                       -- 失败时的错误信息

    CHECK (priority IN ('P0', 'P1', 'P2', 'P3', 'P4')),
    CHECK (status IN ('pending', 'running', 'paused', 'completed', 'failed', 'cancelled'))
);

CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_priority ON tasks(priority);
CREATE INDEX idx_tasks_created ON tasks(created_at DESC);
```

```sql
CREATE TABLE audit_log (
    log_id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增ID，保证顺序
    timestamp       TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 操作时间
    agent_id        TEXT,                               -- 操作者AgentID(或'user'表示创始人)
    operation_type  TEXT NOT NULL,                      -- 操作类型
    target_type     TEXT,                               -- 操作对象类型: task/file/db/agent
    target_id       TEXT,                               -- 操作对象ID
    params          TEXT,                               -- 操作参数(JSON)

CREATE TRIGGER prevent_audit_log_update
BEFORE UPDATE ON audit_log
BEGIN
    SELECT RAISE(ABORT, '审计日志禁止修改');
```

### 3.4 ChromaDB 初始化代码

```python
# memory_store.py - ChromaDB初始化

import chromadb
from chromadb.config import Settings
from pathlib import Path

CHROMA_PATH = Path("data/chromadb")
EMBEDDING_MODEL = "nomic-embed-text"

class MemoryStore:
    def __init__(self):
        CHROMA_PATH.mkdir(parents=True, exist_ok=True)

        self.client = chromadb.PersistentClient(
            path=str(CHROMA_PATH),
            settings=Settings(
                anonymized_telemetry=False,  # 禁用遥测
                allow_reset=True             # 允许重置(开发调试)
            )
        )

        # 初始化4个Collection
        self.collections = {}
        for name in ["agent_main_memory", "agent_audit_memory",
                     "agent_dev_memory", "shared_context"]:
            self.collections[name] = self.client.get_or_create_collection(
                name=name,
                metadata={"hnsw:space": "cosine"}  # 余弦相似度
            )

    def get_collection(self, agent_id: str):
        """获取Agent对应的Collection"""
        mapping = {
            "agent_main": "agent_main_memory",
            "agent_audit": "agent_audit_memory",
            "agent_dev": "agent_dev_memory"
        }
        collection_name = mapping.get(agent_id, "shared_context")
        return self.collections[collection_name]
```

```python
# backup.py - 简化版备份

import shutil
from pathlib import Path
from datetime import datetime

def daily_backup():
    """每日备份SQLite数据库"""
    src = Path("data/solo_ops.db")
    backup_dir = Path("data/backups/daily")
    backup_dir.mkdir(parents=True, exist_ok=True)

    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    dst = backup_dir / f"solo_ops_{timestamp}.db"

    shutil.copy2(src, dst)

    # 清理7天前的备份
    for f in backup_dir.glob("solo_ops_*.db"):
        if (datetime.now() - datetime.fromtimestamp(f.stat().st_mtime)).days > 7:
            f.unlink()

def weekly_backup():
    """每周备份完整data目录"""
    src = Path("data")
    backup_dir = Path("data/backups/weekly")
    backup_dir.mkdir(parents=True, exist_ok=True)

    timestamp = datetime.now().strftime("%Y%m%d")
    dst = backup_dir / f"data_backup_{timestamp}.zip"

    shutil.make_archive(str(dst.with_suffix('')), 'zip', src)

    # 清理4周前的备份
    for f in backup_dir.glob("data_backup_*.zip"):
        file_date = datetime.strptime(f.stem.split('_')[-1], "%Y%m%d")
        if (datetime.now() - file_date).days > 28:
            f.unlink()
```
