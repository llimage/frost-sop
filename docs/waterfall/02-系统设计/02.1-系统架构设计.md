## 1. 设计目标与约束

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| G1 | 支持 3 个常驻 Agent + 9 个角色模板按需实例化，内存峰值 ≤3.5GB |
| G2 | 单页仪表盘实时刷新，页面加载 <2 秒 |
| G3 | 3 个 SOP（DEV-001/002, STR-002）完整执行，阶段状态可追溯 |
| G4 | 所有数据本地存储，API Key 加密，原始数据不出网 |
| G5 | 月度 Token 成本可控，实时显示消耗 |

### 1.2 设计约束

| 约束 | 来源 | 影响 |
|------|------|------|
| 内存 ≤3.5GB | PRD 硬件约束 | Agent 数量、模型加载策略、并发限制 |
| 月度 Token ¥300 | PRD 成本约束 | 本地优先策略、云端调用熔断机制 |
| NiceGUI 前端 | PRD 技术栈 | UI 组件选择、实时刷新机制 |
| SQLite + ChromaDB | PRD 数据层 | 数据模型设计、查询性能、备份策略 |
| Ollama qwen2.5:3b | PRD 本地模型 | 意图解析、简单代码生成、摘要任务 |
| 创始人 IT 小白 | PRD 用户画像 | 零配置启动、错误提示友好、文档完备 |

### 1.3 假设依赖（⚠️ 未验证）

| 假设 ID | 假设内容 | 验证实验 | 设计影响 | 不成立回退 |
|---------|---------|---------|---------|-----------|
| TA-001 | NiceGUI 能做出满意仪表盘 | 实验 1 | 前端框架、UI 组件选型 | 切换 Flutter，重写前端 |
| TA-002 | Ollama 3B 覆盖率 ≥60% | 实验 2 | 本地/云端任务分配比例、成本模型 | 升级模型或增加预算 |
| TA-003 | 3 Agent + Ollama 内存 ≤3.5GB | 实验 3 | Agent 常驻/按需策略、模型加载时机 | 减 Agent 或升级硬件 |

### 2.3 核心设计决策

| # | 决策 | 方案 | 备选方案 | 决策理由 |
|---|------|------|---------|---------|
| 1 | 前端框架 | NiceGUI | Flutter | PRD 技术栈约束，Python 原生 |
| 2 | Agent 常驻策略 | 3 个常驻 + 9 个模板按需 | 12 个常驻 | 内存约束 ≤3.5GB |
| 3 | 模型加载策略 | Ollama 3B 按需加载，用完卸载 | 常驻内存 | 减少内存占用 |
| 4 | 数据访问模式 | 同步阻塞（单用户） | 异步并发 | 单用户系统，简化设计 |
| 5 | Agent 通信 | 内存消息总线（Python Queue） | WebSocket/HTTP | 同进程内通信，零延迟 |
| 6 | 状态持久化 | SQLite 实时写入 | 内存缓存+批量写入 | 单用户，数据量小，实时更可靠 |
| 7 | 错误处理 | 全局异常捕获 + 友好提示 | 分级异常处理 | IT 小白用户，简化错误认知 |
| 8 | 配置管理 | SQLite 表 + 启动加载 | 配置文件 | 零配置启动，UI 内可修改 |

## 3. 模块详细设计

### 3.1 调度器（Scheduler）

**职责**: 系统中枢，连接前端事件与 Agent 执行，管理任务队列和状态广播。

**常驻**: 是（与主 Agent 合并或独立，视内存测试决定）

**核心功能**:
- 接收前端事件（用户输入、按钮点击、配置变更）
- 维护任务队列（P0-P4 优先级）
- 分发任务给主 Agent
- 广播 Agent 状态更新到前端
- 管理 Ollama 模型加载/卸载时机

**状态机**:

```
┌─────────┐    用户输入     ┌─────────┐
│  IDLE   │ ──────────────→ │ PARSING │
│ (空闲)  │                 │ (解析中)│
└─────────┘                 └────┬────┘
     ↑                           │ 意图解析完成
     │                           ▼
     │                      ┌─────────┐
     │    任务完成/失败      │ RUNNING │
     └───────────────────── │ (执行中)│
                            └────┬────┘
                                 │ 需要用户确认
                                 ▼
                            ┌─────────┐
                            │ WAITING │
                            │(等待确认)│
                            └─────────┘
```

**接口**:

```python
class Scheduler:
    def __init__(self, db: Database, memory: MemoryStore, config: Config):
        self.main_agent = MainAgent(db, memory, config)
        self.audit_agent = AuditAgent(db, memory, config)
        self.dev_agent = DevAgent(db, memory, config)
        self.task_queue = PriorityQueue()  # P0-P4
        self.current_task = None
        
    def on_user_input(self, text: str) -> Task:
        """接收用户输入，创建任务，返回任务ID"""
        
    def get_agent_status(self) -> dict:
        """返回3个常驻Agent的当前状态"""
        
    def get_task_result(self, task_id: str) -> dict:
        """返回指定任务的执行结果"""
        
    def on_user_confirm(self, task_id: str, decision: str):
        """用户确认/驳回决策点，继续任务执行"""
```

### 3.2 主 Agent（MainAgent / Orchestrator）

**职责**: 意图解析、SOP 选择、团队组建、任务协调。

**常驻**: 是

**核心功能**:
- 意图解析：调用本地 3B 模型或云端模型，识别用户指令类型
- SOP 匹配：根据意图选择 DEV-001 / DEV-002 / STR-002
- 阶段规划：分解 SOP 为执行阶段，确定串行/并行
- 团队组建：决定需要哪些 Agent 参与（常驻或实例化角色模板）
- 结果汇总：收集各 Agent 输出，整合为最终交付物

**记忆**:
- 长期记忆：历史任务记录、用户偏好、SOP 执行模式
- 短期记忆：当前任务上下文、已执行阶段、待决策点

**接口**:

```python
class MainAgent:
    def __init__(self, db: Database, memory: MemoryStore, config: Config):
        self.memory = memory.get_collection("agent_main_memory")
        
    def parse_intent(self, user_input: str) -> IntentResult:
        """解析用户意图，返回意图类型和置信度"""
        # 优先本地3B，置信度低时 fallback 云端
        
    def select_sop(self, intent: IntentResult) -> SOP:
        """根据意图选择匹配的SOP模板"""
        
    def plan_stages(self, sop: SOP, context: dict) -> list[Stage]:
        """将SOP分解为执行阶段序列"""
        
    def execute_stage(self, stage: Stage, agents: list[Agent]) -> StageResult:
        """执行单个阶段，协调各Agent"""
```

### 3.3 审计 Agent（AuditAgent）

**职责**: 代码审查、成本监督、质量检查、提供技术建议（不裁决）。

**常驻**: 是

**核心功能**:
- 代码审查：检查开发 Agent 输出的代码规范、安全性
- 成本监督：监控 Token 消耗，接近阈值时预警
- 质量检查：检查 SOP 执行结果是否符合验收标准
- 技术建议：对技术方案提供建议，**最终决策权归创始人**

**权限边界**:
- ✅ 可以：审查代码、记录问题、提供建议、预警成本
- ❌ 不可以：自动修改代码、自动拒绝任务、强制停止执行

**接口**:

```python
class AuditAgent:
    def __init__(self, db: Database, memory: MemoryStore, config: Config):
        self.memory = memory.get_collection("agent_audit_memory")
        
    def review_code(self, code: str, language: str) -> ReviewResult:
        """审查代码，返回问题列表（致命/警告/建议）"""
        
    def check_cost(self, current_month_cost: float) -> CostAlert:
        """检查月度成本，返回预警级别（正常/预警/熔断）"""
        
    def review_quality(self, deliverable: str, criteria: list) -> QualityReport:
        """检查交付物质量，返回评分和改进建议"""
        
    def advise_technical(self, proposal: str) -> TechnicalAdvice:
        """对技术方案提供建议，不裁决"""
```

### 3.4 开发 Agent（DevAgent）

**职责**: 代码生成、文件操作、工具执行、单元测试。

**常驻**: 是

**核心功能**:
- 代码生成：根据需求生成 Python/NiceGUI/SQLite 代码
- 文件操作：读写项目文件，遵守路径安全约束
- 工具执行：调用外部工具（Git、pip、pytest 等）
- 单元测试：为生成的代码编写并运行测试

**权限边界**:
- ✅ 可以：读写 `Solo-Ops-Platform/` 目录内文件、运行测试、Git 提交
- ❌ 不可以：删除 `.git/`、修改系统文件、访问 Vault（API Key 存储）

**接口**:

```python
class DevAgent:
    def __init__(self, db: Database, memory: MemoryStore, config: Config):
        self.memory = memory.get_collection("agent_dev_memory")
        self.tools = ToolKit()
        
    def generate_code(self, requirement: str, language: str) -> CodeResult:
        """根据需求生成代码"""
        
    def run_tests(self, code: str, test_cases: list) -> TestResult:
        """运行单元测试，返回通过/失败详情"""
        
    def execute_tool(self, tool_name: str, params: dict) -> ToolResult:
        """执行指定工具，返回结果"""
```

### 3.5 SOP 引擎（SOPEngine）

**职责**: SOP 模板管理、阶段执行、状态流转、决策点管理。

**常驻**: 否（由主 Agent 调用）

**核心功能**:
- SOP 模板加载：从磁盘加载 3 个 SOP 的 JSON/YAML 定义
- 阶段执行：按定义的顺序执行每个阶段
- 状态流转：管理阶段状态（pending → running → completed / failed）
- 决策点：遇到需要创始人确认的阶段，暂停并通知前端
- 重试机制：阶段失败时自动重试 1 次

**SOP 模板格式**:

```yaml
sop_id: "DEV-001"
name: "新功能开发"
version: "1.0"
stages:
  - stage_id: "s1"
    name: "需求讨论"
    mode: "parallel"  # parallel / serial
    agents: ["product_manager", "architect"]  # 角色模板名称
    input: "原始需求描述"
    output: "PRD文档 + 技术方案"
    decision_point: true  # 是否需要创始人确认
    decision_options: ["确认", "驳回"]
    token_estimate: 800
    
  - stage_id: "s2"
    name: "设计评审"
    mode: "serial"
    agents: ["architect", "developer"]
    input: "阶段1输出物"
    output: "详细设计 + 接口定义"
    decision_point: true
    token_estimate: 600
    
  - stage_id: "s3"
    name: "开发实现"
    mode: "serial"
    agents: ["developer"]
    input: "阶段2输出物"
    output: "代码 + 单元测试"
    decision_point: false
    token_estimate: 1200
    
  - stage_id: "s4"
    name: "测试验证"
    mode: "serial"
    agents: ["tester"]
    input: "阶段3输出物"
    output: "测试报告"
    decision_point: false
    token_estimate: 400
    
  - stage_id: "s5"
    name: "审计封板"
    mode: "serial"
    agents: ["auditor"]
    input: "阶段4输出物"
    output: "审计报告"
    decision_point: true
    token_estimate: 200

total_token_estimate: 2200
priority: "P1"
```

**接口**:

```python
class SOPEngine:
    def __init__(self, sop_dir: str):
        self.sops = self._load_sops(sop_dir)
        
    def load_sop(self, sop_id: str) -> SOP:
        """加载指定SOP模板"""
        
    def execute_sop(self, sop: SOP, context: dict) -> SOPResult:
        """执行完整SOP，返回结果和状态历史"""
        
    def execute_stage(self, stage: Stage, context: dict) -> StageResult:
        """执行单个阶段"""
        
    def pause_at_decision(self, stage: Stage, question: str) -> DecisionPoint:
        """在决策点暂停，等待用户确认"""
```

### 3.6 工具框架（ToolKit）

**职责**: 统一管理 Agent 可调用的工具，权限校验，调用审计。

**常驻**: 否（各 Agent 实例化时使用）

**核心工具列表**:

| 工具名 | 功能 | 权限级别 | 适用 Agent |
|--------|------|---------|-----------|
| `file_read` | 读取文件内容 | P1 | 所有 |
| `file_write` | 写入文件内容 | P2 | 开发 Agent |
| `file_list` | 列出目录文件 | P1 | 所有 |
| `code_execute` | 执行 Python 代码 | P3 | 开发 Agent |
| `git_commit` | Git 提交代码 | P2 | 开发 Agent |
| `ollama_chat` | 调用本地 3B 模型 | P1 | 所有 |
| `api_chat` | 调用云端 LLM API | P1 | 所有（受成本熔断限制） |
| `db_query` | SQLite 查询 | P1 | 所有 |
| `db_insert` | SQLite 插入 | P2 | 所有 |
| `vector_search` | ChromaDB 向量检索 | P1 | 所有 |
| `vector_insert` | ChromaDB 向量插入 | P2 | 所有 |
| `cost_check` | 检查当前成本 | P1 | 审计 Agent |

**权限级别**（与02.5安全设计一致，采用P0-P4五级模型）:
- P0: 创始人专用（系统关机、配置修改、数据删除）
- P1: 读取权限（文件读取、数据库查询、日志查看、LLM调用）
- P2: 写入权限（文件写入、数据库修改、Git提交）
- P3: 执行权限（代码执行、命令执行）
- P4: 管理权限（Agent启停、配置变更、权限分配）

**工具权限映射**:
| 工具 | 权限 | 说明 |
|------|------|------|
| file_read | P1 | 读取操作 |
| file_write | P2 | 写入操作 |
| file_list | P1 | 读取操作 |
| code_execute | P3 | 代码执行 |
| git_commit | P2 | 写入操作 |
| ollama_chat | P1 | 读取/查询 |
| api_chat | P1 | 读取/查询（受成本熔断限制） |
| db_query | P1 | 读取操作 |
| db_insert | P2 | 写入操作 |
| vector_search | P1 | 读取操作 |
| vector_insert | P2 | 写入操作 |
| cost_check | P1 | 读取操作 |

**接口**:

```python
class ToolKit:
    def __init__(self, db: Database, config: Config):
        self.tools = self._register_tools()
        self.audit_log = db.audit_log
        
    def register_tool(self, name: str, func: callable, permission: str, agents: list):
        """注册工具，指定权限和可用Agent"""
        
    def call_tool(self, name: str, agent_id: str, params: dict) -> ToolResult:
        """调用工具，先校验权限，再执行，最后记录审计日志"""
        
    def list_tools(self, agent_id: str) -> list[ToolInfo]:
        """返回指定Agent可用的工具列表"""
```

### 3.7 记忆管理（MemoryStore）

**职责**: 管理 Agent 专属记忆、向量检索、上下文注入。

**常驻**: 是（作为服务被各 Agent 调用）

**核心功能**:
- 专属记忆：每个常驻 Agent 独立的 ChromaDB Collection
- 向量检索：基于 nomic-embed-text 的语义检索
- 上下文注入：任务执行时自动检索相关记忆，注入提示词
- 记忆写入：Agent 执行结果自动写入记忆库

**存储结构**:

```
ChromaDB Collections:
  ├── agent_main_memory      # 主Agent记忆
  ├── agent_audit_memory     # 审计Agent记忆
  ├── agent_dev_memory       # 开发Agent记忆
  └── shared_context         # 共享上下文（任务级）
```

**接口**:

```python
class MemoryStore:
    def __init__(self, chroma_path: str, embedding_model: str):
        self.client = chromadb.PersistentClient(path=chroma_path)
        self.embedder = OllamaEmbedding(model=embedding_model)
        
    def get_collection(self, agent_id: str) -> Collection:
        """获取或创建Agent专属Collection"""
        
    def add_memory(self, agent_id: str, text: str, metadata: dict):
        """向指定Agent记忆库添加记录"""
        
    def search_memory(self, agent_id: str, query: str, top_k: int = 5) -> list[Memory]:
        """语义检索Agent记忆"""
        
    def get_context_for_task(self, agent_id: str, task_description: str, max_tokens: int = 1000) -> str:
        """为任务检索相关记忆，格式化为上下文字符串"""
```

### 3.8 成本追踪（CostTracker）

**职责**: 实时记录和监控 Token 消耗，预警和熔断。

**常驻**: 是（轻量级服务）

**核心功能**:
- 记录每次 LLM 调用：模型、输入 Token、输出 Token、成本
- 月度汇总：按 Agent、SOP、任务维度汇总
- 实时显示：仪表盘显示当前月度消耗
- 预警机制：80% 预警（橙色），100% 熔断（红色，暂停非 P0 任务）

**接口**:

```python
class CostTracker:
```

## 5. Agent 架构设计

### 5.1 Agent 基类设计

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional

@dataclass
class AgentConfig:
    agent_id: str
    name: str
    role: str
    model_preference: str  # "local" / "cloud" / "auto"
    memory_collection: str
    available_tools: list[str]

class BaseAgent(ABC):
    def __init__(self, config: AgentConfig, db: Database, memory: MemoryStore, tools: ToolKit):
        self.config = config
        self.db = db
        self.memory = memory.get_collection(config.memory_collection)
        self.tools = tools
        self.status = "idle"  # idle / running / paused / error
        self.current_task = None
        
    @abstractmethod
    def execute(self, task: Task) -> AgentResult:
        """执行分配的任务"""
        pass
        
    def update_status(self, status: str, progress: Optional[int] = None):
        """更新Agent状态，广播到前端"""
        self.status = status
        self.db.update_agent_status(self.config.agent_id, status, progress)
        
    def recall_memory(self, query: str, top_k: int = 3) -> str:
        """检索相关记忆，格式化为上下文"""
        memories = self.memory.search(query=query, n_results=top_k)
        return self._format_memories(memories)
        
    def save_memory(self, content: str, metadata: dict):
        """保存执行结果到记忆库"""
        self.memory.add(documents=[content], metadatas=[metadata])
```

### 5.3 Agent 间通信协议

MVP 阶段简化设计：**3 个常驻 Agent 通过调度器中转通信**，不直接点对点。

```python
# 通信消息格式
@dataclass
class AgentMessage:
    msg_id: str
    from_agent: str
    to_agent: str
    msg_type: str  # "request" / "response" / "notify" / "advice"
    topic: str
    content: str
    timestamp: float
    task_id: Optional[str] = None

# 通信流程示例：开发Agent请求审计Agent审查代码
1. 开发Agent → 调度器: 
   AgentMessage(
       from_agent="dev_agent",
       to_agent="audit_agent", 
       msg_type="request",
       topic="code_review",
       content="请审查这段代码...[代码]",
       task_id="task_001"
   )

2. 调度器 → 审计Agent: 转发消息

3. 审计Agent → 调度器:
   AgentMessage(
       from_agent="audit_agent",
       to_agent="dev_agent",
       msg_type="response",
       topic="code_review_result",
       content="审查结果：1个警告...建议...",
       task_id="task_001"
   )

4. 调度器 → 开发Agent: 转发回复

5. 调度器: 记录通信到审计日志
```

## 7. 风险与回退方案

### 7.1 架构级风险

| 风险 | 概率 | 影响 | 回退方案 |
|------|------|------|---------|
| NiceGUI 性能不足（单页大量组件卡顿） | 中 | 高 | 减少实时刷新频率（5秒→30秒），或切 Flutter |
| Ollama 3B 意图解析准确率 <60% | 中 | 高 | 增加云端 fallback 比例，调整成本模型 |
| 3 个 Agent + Ollama 内存 >3.5GB | 中 | 高 | 审计 Agent 改为按需加载，仅主+开发常驻 |
| SQLite 并发写入冲突 | 低 | 中 | 增加写入队列，串行化数据库操作 |
| ChromaDB 向量检索 >1 秒 | 低 | 中 | 减少 Collection 数据量，增加索引优化 |

### 7.2 设计决策回退

| 决策 | 当前方案 | 回退条件 | 回退方案 |
|------|---------|---------|---------|
| 前端框架 | NiceGUI | 实验1不通过 | 评估 Flutter，预计增加 2-4 周 |
| Agent 常驻 | 3 个常驻 | 实验3内存不足 | 2 个常驻（主+开发），审计按需 |
| 模型策略 | 3B 本地优先 | 实验2覆盖率<40% | 升级 7B 或增加云端预算 |
| 通信机制 | 调度器中转 | 延迟 >2 秒 | Agent 直接通信（共享内存） |
| 数据访问 | 同步阻塞 | 性能不足 | 增加内存缓存层 |
