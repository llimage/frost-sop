## 1. 设计原则

### 1.1 核心原则

| 原则 | 说明 | 设计体现 |
|------|------|---------|
| **创始人优先** | 从配置读取创始人姓名，默认为"老板" | 所有操作3步内可达，无学习成本 |
| **暗色主题** | 长时间使用，护眼 | 全局暗色，高对比度文字 |
| **信息密度适中** | 单屏展示关键信息 | 仪表盘一屏可见所有Agent状态 |
| **实时反馈** | 操作后立即看到结果 | Agent状态实时更新，日志实时滚动 |
| **中文优先** | 创始人IT小白 | 所有界面中文，技术术语有提示 |
| **成本可见** | Token消耗透明 | 每个Agent卡片显示累计成本 |

### 1.2 设计约束

| 约束 | 来源 | 影响 |
|------|------|------|
| NiceGUI 组件库 | 技术选型 | 使用原生组件，不引入外部CSS框架 |
| 单用户桌面端 | PRD | 无需响应式移动端适配 |
| 暗色主题唯一 | PRD | 不提供主题切换 |
| 创始人IT小白 | PRD | 避免技术术语，提供操作提示 |
| 内存≤3.5GB | PRD | 避免复杂动画，减少DOM节点 |

**设计决策**: MVP 采用顶部导航栏 + 下拉菜单，而非侧边栏。原因：
- 页面数量少（8个），侧边栏浪费空间
- 创始人主要使用仪表盘，其他页面低频访问
- NiceGUI 的 `ui.header()` 实现简单

## 8. 响应式与适配

### 8.1 适配策略

MVP 仅支持**桌面端**，最小分辨率 **1280×720**。

| 分辨率 | 布局调整 |
|--------|---------|
| ≥1920×1080 | 标准布局，Agent卡片3列 |
| 1280×720 ~ 1920×1080 | 标准布局，Agent卡片2列 |
| <1280×720 | 提示"请使用更大屏幕" |

### 8.2 性能优化

| 优化项 | 措施 | 原因 |
|--------|------|------|
| 定时刷新 | 3秒轮询，非实时WebSocket | 减少CPU占用 |
| 日志限制 | 最多显示100条，滚动加载 | 防止内存泄漏 |
| 动画限制 | 仅状态灯使用CSS动画 | 减少GPU占用 |
| 图片限制 | 不使用图片，仅用emoji和图标 | 减少内存 |
| 组件复用 | Agent卡片使用统一组件 | 减少DOM节点 |
