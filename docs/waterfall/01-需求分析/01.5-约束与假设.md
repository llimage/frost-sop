## 1. 技术约束

### 1.1 前端框架约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| TC-001 | 必须使用 NiceGUI | 创始人已确认放弃 Streamlit，NiceGUI 为 Python 原生、Vue 级美观 | 学习曲线高于 Streamlit，社区资源较少 | 预留 Flutter 备选方案；编写详细 NiceGUI 教程文档 |
| TC-002 | 暗色主题为默认 | 创始人偏好；长时间使用护眼 | 亮色主题适配工作量 | 主题系统抽象层，切换仅改 CSS 变量 |
| TC-003 | 响应式布局，最小 1280×720 | 创始人主要使用笔记本 | 小屏幕设备体验下降 | 核心功能在 1280×720 可完整使用，大屏增强 |
| TC-004 | 支持桌面应用模式 | 创始人习惯本地应用，非浏览器标签 | 打包复杂度增加 | 使用 NiceGUI 原生桌面模式或 PyInstaller |

### 1.2 后端技术约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| TC-005 | Python 3.10+ | 创始人唯一熟悉的编程语言；NiceGUI 要求 | 性能低于编译型语言 | 关键路径用 C 扩展或缓存优化 |
| TC-006 | SQLite 为结构化数据库 | 零配置、单文件、易备份 | 并发写入性能有限 | 单用户系统，串行写入足够；WAL 模式启用 |
| TC-007 | ChromaDB 为向量数据库 | 本地优先、Python 原生、易集成 | 功能不如 Pinecone/Weaviate 丰富 | 仅使用核心向量检索功能，高级功能自研 |
| TC-008 | Ollama 为本地模型运行时 | 零配置、模型管理方便 | 仅支持特定模型格式 | 仅使用 Ollama 官方支持的模型 |

### 1.3 AI 模型约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| TC-009 | 本地模型：qwen2.5:3b | 2GB 大小，4GB 内存可跑，中文优化 | 能力有限，复杂任务需云端 | 知识晶体+模板填充补偿；复杂任务自动降级到云端 |
| TC-010 | Embedding 模型：nomic-embed-text | 100MB，多语言支持 | 中文语义理解不如专用模型 | 结合关键词检索作为补充 |
| TC-011 | 云端模型：Kimi/Coze/DeepSeek | 创始人已有账号，中文能力强 | 有 Token 成本和网络依赖 | 熔断机制；本地优先策略 |
| TC-012 | 月度 Token 预算 ¥300 | 创始人明确约束 | 功能复杂度受限 | 成本优化：知识晶体、模板填充、上下文压缩 |

### 1.4 部署约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| TC-013 | 纯本地部署，数据不出门 | 产品宣言核心原则 | 无法使用云端服务增强 | 本地模型+本地数据库+本地文件系统 |
| TC-014 | 无需服务器/域名/SSL | 创始人无运维能力 | 无法提供 Web 访问给外部 | 仅桌面应用，不对外服务 |
| TC-015 | 单用户系统 | 创始人自用，非 SaaS | 无需用户认证、权限系统简化 | 本地文件系统权限即用户隔离 |

## 2. 业务约束

### 2.1 创始人能力约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| BC-001 | 创始人是 IT 小白 | 技术栈以低代码/无代码为主 | 复杂配置无法完成 | 零配置启动；所有配置有默认值；错误提示用中文 |
| BC-002 | 创始人极度社恐 | 排斥 1V1 服务，倾向内容产品 | 不需要客服、协作功能 | 产品设计聚焦单人使用 |
| BC-003 | 创始人怕失败 | 追求秩序感，需要确定性 | 频繁变更会焦虑 | 瀑布式开发；版本发布有明确计划；变更需审批 |
| BC-004 | 创始人记不住文档 | 很多文档有用，很多过时 | 需要自动提炼和过期清理 | 知识晶体生命周期自动管理 |

### 2.2 商业约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| BC-005 | 非 SaaS，不卖钱 | 情怀作品，自用为主 | 无收入，无法投入商业资源 | 控制开发成本，优先使用开源方案 |
| BC-006 | 三个核心项目并行 | 心域探险/心栖陪伴/心理歌曲 | 系统需支持多项目隔离 | 项目级数据隔离；Agent 记忆隔离 |
| BC-007 | 2 年内月均 30,000 元被动收入 | 商业目标 | 内容营销功能优先级高 | MKT-001/002/003 在 P1 优先级 |

### 2.3 法律合规约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| BC-008 | 个体户资质 | 无法注册公司，部分平台受限 | 某些平台 API 需企业资质 | 优先支持个人可使用的平台 |
| BC-009 | 心理咨询内容合规 | 涉及心理健康，需避免医疗建议 | 内容审核要求严格 | 审计 Agent 检查敏感词；免责声明模板 |
| BC-010 | 数据隐私法规 | 即使本地部署，需符合法规 | 用户数据处理需有依据 | 数据主权设计；用户完全控制数据 |

## 3. 时间约束

| 约束 ID | 约束描述 | 原因 | 影响 | 缓解措施 |
|---------|---------|------|------|---------|
| TM-001 | 不设定硬截止日期 | 创始人：宁可花点时间也要弄满意 | 无法按日期倒排计划 | 按瀑布阶段推进，每阶段验收通过后再进入下一阶段 |
| TM-002 | 瀑布式开发，阶段间不回头 | 创始人对敏捷结果不满意 | 需求变更成本高 | 需求冻结机制；变更需创始人+审计 Agent 共同确认 |
| TM-003 | 创始人时间碎片化 | 心理咨询师，咨询时间不可控 | 开发沟通时间不固定 | 异步文档沟通为主；关键节点预约时间 |

## 5. 关键假设

### 5.1 技术假设

| 假设 ID | 假设描述 | 验证方式 | 验证时间 | 不成立的影响 | 回退方案 |
|---------|---------|---------|---------|-------------|---------|
| TA-001 | NiceGUI 能满足所有 UI 需求 | 原型验证：实现仪表盘核心页面 | 设计阶段 | 需切换 Flutter，前端重写 | 预留 Flutter 方案，NiceGUI 不顺手时切换 |
| TA-002 | Ollama qwen2.5:3b 能覆盖 80% 本地场景 | 基准测试：用 50 个典型任务测试 | 设计阶段 | 被迫增加云端调用，超预算 | 优化知识晶体和模板；接受部分任务简化 |
| TA-003 | ChromaDB 本地性能满足需求 | 压力测试：10,000 条向量检索 <1 秒 | 设计阶段 | 需切换其他向量库 | 评估 Milvus Lite 或 Faiss |
| TA-004 | SQLite WAL 模式能支撑单用户并发 | 测试：同时读写 100 次无冲突 | 设计阶段 | 需引入数据库锁或队列 | 实现简单的任务队列 |
| TA-005 | NiceGUI 桌面模式打包可行 | 原型验证：PyInstaller 打包成功运行 | 实现阶段 | 需改为浏览器模式 | 接受浏览器模式，绑定 127.0.0.1 |

### 5.2 业务假设

| 假设 ID | 假设描述 | 验证方式 | 验证时间 | 不成立的影响 | 回退方案 |
|---------|---------|---------|---------|-------------|---------|
| BA-001 | 创始人能学会 NiceGUI 基础操作 | 培训文档 + 手把手指导后测试 | 实现阶段 | 前端改动需完全依赖 AI Agent | 增加更多自动化配置 |
| BA-002 | 三个项目（心域/心栖/心理歌曲）能共用同一套 SOP | 每个项目至少运行一次核心 SOP | 测试阶段 | 需为每个项目定制 SOP | 项目级 SOP 覆盖机制 |
| BA-003 | 内容营销渠道（小红书/抖音/视频号）API 或手动发布可行 | 验证各平台发布方式 | 实现阶段 | 某些渠道无法自动化 | 标记为"仅手动"，不实现 API |
| BA-004 | 创始人愿意定期记录能量状态 | 观察使用数据 | 发布后 1 个月 | 能量感知调度失效 | 改为基于任务完成率推断 |

### 5.3 外部依赖假设

| 假设 ID | 假设描述 | 验证方式 | 验证时间 | 不成立的影响 | 回退方案 |
|---------|---------|---------|---------|-------------|---------|
| EA-001 | Ollama 持续维护且支持 qwen2.5:3b | 检查 Ollama 官方模型库 | 持续 | 需手动下载和配置模型 | 使用其他本地运行时（llama.cpp） |
| EA-002 | DeepSeek/Kimi API 稳定且价格合理 | 监控 API 可用性和价格 | 持续 | 需切换其他提供商 | 支持多提供商配置，自动切换 |
| EA-003 | Python 3.10+ 在目标平台稳定运行 | 在 Windows/macOS/Linux 测试 | 设计阶段 | 需降级 Python 版本 | 评估最低支持版本 |
| EA-004 | NiceGUI 社区持续活跃 | 检查 GitHub 提交频率和 Issue 响应 | 持续 | 遇到 Bug 无人修复 |  fork 并自行修复；或切换 Flutter |
