# 01.2 用户场景与用例

> **文档类型**: 瀑布式需求分析 —— 用户场景与用例
> **版本**: V1.0
> **状态**: 需求冻结
> **前置文档**: 01.1-项目背景与目标.md

---

## 1. 用户角色定义

S-O-P 只有一个用户：**创始人（廖亮）**。所有设计围绕单一用户优化。

| 角色 | 描述 | 使用频率 | 核心诉求 |
|------|------|---------|---------|
| 创始人 | 一人公司所有者，心理咨询师 | 每日 | 有序、可视、有层次、有驱动 |

---

## 2. 核心场景总览

```
场景组 A: 日常运营（每日）
  ├── A1: 早晨启动 —— 查看今日任务和日程
  ├── A2: 任务执行 —— 向主 Agent 下达指令
  ├── A3: 状态记录 —— 记录当前能量/情绪状态
  └── A4: 晚间收尾 —— 查看今日完成、明日预告

场景组 B: 项目开发（按需）
  ├── B1: 新功能立项 —— 从想法到 PRD
  ├── B2: 功能开发 —— Agent 协作编码
  ├── B3: Bug 修复 —— 紧急响应流程
  └── B4: 技术债务清理 —— 定期维护

场景组 C: 内容营销（每周）
  ├── C1: 内容选题 —— 从需求池到母酱
  ├── C2: 多渠道发布 —— 母酱 → 各平台适配
  └── C3: 反馈处理 —— 用户评论/私信响应

场景组 D: 运营行政（每月/每季度）
  ├── D1: 财务月结 —— 收入支出核算
  ├── D2: 合规审计 —— 法律/税务/平台合规检查
  ├── D3: 数据备份 —— 3-2-1 备份策略
  ├── D4: 合同管理 —— 供应商/合作方合同跟踪
  ├── D5: 安全审计 —— 密码/密钥/资产检查
  └── D6: 知识资产管理 —— 文档 → 晶体生命周期

场景组 E: 战略管理（每季度）
  ├── E1: 项目立项 —— 新项目可行性评估
  ├── E2: 周期性回顾 —— 日/周/月/季回顾
  └── E3: 风险预警 —— 自动监测与应对
```

## 3. 场景详细用例

### 场景组 A: 日常运营

#### A1: 早晨启动

**触发**: 创始人每天早晨打开 S-O-P
**前置条件**: 系统已启动，秘书 Agent 已加载今日日程
**主流程**:

1. 系统显示仪表盘首页
2. 顶部显示今日日期和问候语
3. 左侧显示「今日任务」列表（按优先级排序）
4. 右侧显示「今日日程」时间线（秘书 Agent 记录 + 项目依赖）
5. 显示「能量状态」输入器（创始人可快速选择当前能量水平 0-100%）
6. 显示「紧急事项」面板（如有 P0 任务未处理）
7. 创始人可选择：
   - 点击任务开始执行 → 进入 A2
   - 记录能量状态 → 系统根据能量调整任务建议
   - 查看项目总览 → 横向视图

**异常流程**:
- 如有 P0 任务未处理 → 弹窗提醒，必须确认后才可进入其他操作
- 如能量 < 30% → 系统建议只做低强度任务，高优先级任务延后

**后置条件**: 创始人了解今日全貌，可开始工作
**涉及 SOP**: 无（系统级功能）

---

#### A2: 任务执行

**触发**: 创始人向主 Agent 下达指令
**前置条件**: 系统已启动，主 Agent 在线
**主流程**:

1. 创始人在 CEO 面板输入指令，例如：
   - "帮我分析心域探险的竞品"
   - "写一份小红书文案，主题是情绪管理"
   - "修复昨天发现的登录 Bug"
2. 主 Agent 解析意图，识别任务类型
3. 主 Agent 选择对应 SOP，组建 Agent 团队
4. 系统显示「任务执行中」状态，纵向流程可视化：
   - 当前阶段名称
   - 正在执行的 Agent
   - 已完成阶段（绿色 ✓）
   - 待执行阶段（灰色）
5. 各 Agent 按 SOP 阶段协作执行
6. 每完成一个阶段，系统更新进度
7. 需要创始人决策时，系统弹出确认框
8. 全部阶段完成后，系统显示结果
9. 创始人可选择：
   - 确认完成 → 任务归档
   - 要求修改 → 返回对应阶段
   - 保存到知识库 → 触发 OPS-006

**异常流程**:
- 主 Agent 无法识别意图 → 提示创始人澄清，提供选项
- 执行中遇到致命错误 → 审计 Agent 介入，暂停任务，通知创始人
- Token 预算即将耗尽 → 系统预警，建议切换本地模型或暂停非 P0 任务

**后置条件**: 任务完成或暂停，结果可查看/下载/归档
**涉及 SOP**: 根据任务类型自动选择（DEV-001 / MKT-001 / 等）

---

#### A3: 状态记录

**触发**: 创始人主动记录，或系统定时提醒
**前置条件**: 系统已启动
**主流程**:

1. 创始人点击「记录状态」按钮
2. 系统弹出状态记录面板：
   - 能量水平（0-100% 滑动条）
   - 情绪状态（选择：专注 / 疲惫 / 焦虑 / 平静 / 兴奋 / 低落）
   - 备注（可选文本）
3. 创始人提交
4. 系统保存状态到 SQLite
5. 系统分析状态与当前任务的匹配度：
   - 能量高 + 复杂任务 → 推荐继续
   - 能量低 + 复杂任务 → 推荐切换简单任务或休息
6. 秘书 Agent 记录到日程

**后置条件**: 状态数据可用于后续任务推荐和周期性回顾
**涉及 SOP**: 无（系统级功能）

---

#### A4: 晚间收尾

**触发**: 创始人点击「今日回顾」或系统定时（21:00）提醒
**前置条件**: 当日有任务执行记录
**主流程**:

1. 系统显示「今日回顾」面板：
   - 完成任务数 / 计划任务数
   - 消耗 Token 数 / 预算
   - 能量状态变化曲线
   - Agent 活动统计
2. 系统显示「明日预览」：
   - 明日日程（秘书 Agent 记录 + 项目依赖）
   - 建议优先任务
3. 创始人可选择：
   - 确认 → 关闭回顾
   - 记录反思 → 保存到知识库
   - 调整明日计划 → 修改秘书 Agent 日程

**后置条件**: 当日数据归档，明日计划就绪
**涉及 SOP**: STR-002（周期性回顾，简化版）

---

### 场景组 B: 项目开发

#### B1: 新功能立项

**触发**: 创始人有新想法，想启动一个新功能
**前置条件**: 系统已启动，STR-001 SOP 已加载
**主流程**:

1. 创始人向主 Agent 输入："我想给心域探险加一个情绪日记功能"
2. 主 Agent 识别为「项目立项」，触发 STR-001 SOP
3. 阶段 1: 创始人提供立项需求描述（已包含在指令中）
4. 阶段 2: 产品经理 Agent + 架构师 Agent + 营销 Agent 并行可行性评估
   - 市场维度：竞品分析、用户需求
   - 技术维度：实现难度、技术选型
   - 商业维度：与现有产品协同、变现潜力
   - 风险维度：技术风险、合规风险
5. 系统显示可行性报告，创始人审阅
6. 阶段 3: 主 Agent 资源规划
   - 时间估算
   - 所需 Agent 团队
   - Token 预算估算
   - 与现有项目的冲突提示
7. 系统显示资源规划方案，创始人确认
8. 阶段 4: 创始人决策
   - 确认 → 项目启动，进入 DEV-001
   - 驳回 → 入「想法库」，后续可重新激活
   - 调整 → 返回阶段 2

**异常流程**:
- 可行性评估发现重大技术障碍 → 架构师 Agent 标记风险，创始人决策是否继续
- 资源规划发现与现有项目冲突 → 主 Agent 提示冲突，创始人调整优先级

**后置条件**: 新项目立项完成，或入想法库
**涉及 SOP**: STR-001

---

#### B2: 功能开发

**触发**: 立项确认后，或创始人直接下达开发指令
**前置条件**: 需求已明确，DEV-001 SOP 已加载
**主流程**:

1. 阶段 1: 需求讨论（并行）
   - 产品经理 Agent + 架构师 Agent
   - 输入：创始人原始需求
   - 输出：PRD 文档 + 技术方案
   - 决策：创始人确认 → 进阶段 2；驳回 → 返回阶段 1
2. 阶段 2: 设计评审（串行）
   - 架构师 Agent + 开发 Agent
   - 输入：阶段 1 输出物
   - 输出：详细设计 + 接口定义
   - 决策：审计 Agent 技术评审通过 → 进阶段 3；不通过 → 返回阶段 2
3. 阶段 3: 开发实现（串行）
   - 开发 Agent
   - 输入：阶段 2 输出物
   - 输出：代码 + 单元测试
   - 决策：代码自检通过 → 进阶段 4
4. 阶段 4: 测试验证（串行）
   - 测试 Agent
   - 输入：阶段 3 输出物
   - 输出：测试报告
   - 决策：测试通过 → 进阶段 5；失败 → 返回阶段 3
5. 阶段 5: 审计封板（串行）
   - 审计 Agent
   - 输入：阶段 4 输出物
   - 输出：审计报告（致命 / 警告 / 建议）
   - 决策：致命 = 0 → 交付创始人确认上线；致命 > 0 → 驳回返回阶段 3

**异常流程**:
- 阶段 2 审计不通过 → 返回阶段 2，架构师 Agent 修改设计
- 阶段 4 测试失败 → 返回阶段 3，开发 Agent 修复
- 阶段 5 发现致命问题 → 返回阶段 3，开发 Agent 修复
- 创始人中途要求暂停 → 保存状态，稍后恢复

**后置条件**: 功能开发完成，可上线或归档
**涉及 SOP**: DEV-001

---

#### B3: Bug 修复

**触发**: 测试 Agent 发现 Bug、用户反馈、或创始人直接报告
**前置条件**: Bug 报告已录入系统，DEV-002 SOP 已加载
**主流程**:

1. 阶段 1: 紧急分级（串行）
   - 主 Agent 自动分级
   - 输入：Bug 报告
   - 输出：P0 致命 / P1 严重 / P2 一般
2. 阶段 2: 定位分析（并行）
   - 开发 Agent + 架构师 Agent
   - 输入：阶段 1 分级报告
   - 输出：根因分析报告
   - 决策：根因明确 → 进阶段 3；根因不明 → 创始人决策
3. 阶段 3: 修复验证（串行）
   - 开发 Agent → 测试 Agent
   - 输入：阶段 2 根因报告
   - 输出：修复代码 + 回归测试报告
   - 决策：测试通过 → 进阶段 4；失败 → 返回阶段 3
4. 阶段 4: QA 建档（串行）
   - 审计 Agent
   - 输出：QA 记录（症状 / 根因 / 修复 / 验证 / 影响范围）
5. 阶段 5: 教训提取（串行）
   - 审计 Agent
   - 输出：教训记录（踩坑点 / 解决方案 / 关联 SOP）
6. 阶段 6: 审计封板（串行）
   - 审计 Agent
   - 输出：快速审计报告
   - 决策：致命 = 0 → 交付创始人确认上线；致命 > 0 → 驳回返回阶段 3

**异常流程**:
- P0 Bug → 立即抢占所有 Agent 资源，创始人必须立即知情
- P1 Bug → 插入当前队列最前
- P2 Bug → 排队等待，不中断当前工作
- 同类型踩坑 ≥3 次 → 触发教训归集 → 审计 Agent 提炼 SOP 补丁 → 自动生效（创始人事后知情）

**后置条件**: Bug 修复完成，QA 记录归档，教训入库
**涉及 SOP**: DEV-002

---

#### B4: 技术债务清理

**触发**: 每周定时（周日 2:00）+ 版本发布后强制
**前置条件**: 审计 Agent 已配置定期扫描，DEV-003 SOP 已加载
**主流程**:

1. 阶段 1: 债务识别（自动）
   - 审计 Agent 定期扫描
   - 输出：债务清单（警告累积 / 重复代码 / 过时依赖 / 性能瓶颈）
2. 阶段 2: 影响评估（并行）
   - 架构师 Agent + 开发 Agent
   - 输出：影响评估报告（清理成本 / 风险 / 收益）
3. 阶段 3: 清理执行（串行）
   - 开发 Agent
   - 输出：清理代码 + 变更日志
   - 规则：按优先级分批，不中断当前项目工作
4. 阶段 4: 验证审计（串行）
   - 测试 Agent + 审计 Agent
   - 输出：回归测试报告 + 审计报告
   - 决策：致命 = 0 → 自动合并；致命 > 0 → 自动回滚

**异常流程**:
- 清理导致问题 → 自动回滚到清理前版本
- 与当前项目冲突 → 延后到项目完成后

**后置条件**: 技术债务清理完成，或延后
**涉及 SOP**: DEV-003

---

### 场景组 C: 内容营销

#### C1: 内容选题

**触发**: 创始人需要新内容，或营销 Agent 定时推荐
**前置条件**: MKT-003 SOP 已加载
**主流程**:

1. 阶段 1: 选题策划（并行）
   - 营销 Agent + 内容 Agent
   - 来源：用户反馈需求池、热点缓存、历史爆款模式、创始人直接输入
   - 输出：选题池（3-5 个候选 + 优先级）
   - 规则：本地知识图谱查询为主，20% 选题需大模型创意突破
2. 阶段 2: 内容创作（串行）
   - 内容 Agent
   - 输出：母酱（核心内容）
3. 阶段 3: 内部评审（并行）
   - 营销 Agent + 审计 Agent
   - 检查：品牌调性、转化潜力、合规、敏感词
   - 输出：评审报告
4. 阶段 4: 创始人确认（人工）
   - 创始人决策：确认 → 进入 MKT-001 渠道适配；修改 → 返回阶段 2

**后置条件**: 母酱内容确认，可进入多渠道适配
**涉及 SOP**: MKT-003

---

#### C2: 多渠道发布

**触发**: 母酱内容确认后
**前置条件**: MKT-001 SOP 已加载，渠道配置已定义
**主流程**:

1. 阶段 1: 母酱生产（串行）
   - 内容 Agent
   - 输入：创始人选题命令或项目计划
   - 输出：核心内容（完整、深度、不针对任何平台）
   - 决策：创始人确认 → 进阶段 2；驳回 → 重写
2. 阶段 2: 渠道适配（并行）
   - 内容 Agent × N
   - 输入：阶段 1 母酱 + 渠道配置
   - 输出：各渠道定制版（小红书 / 抖音 / 视频号 / 公众号 / 头条 / 微博）
   - 规则：本地模板填充为主，仅 1-2 个渠道需大模型微调
3. 阶段 3: 创始人确认（人工）
   - 创始人确认 / 修改意见（可按渠道单独确认）
   - 决策：确认 → 进阶段 4；某渠道修改 → 该渠道返回阶段 2
4. 阶段 4: 发布审计（并行）
   - 审计 Agent × N
   - 输入：阶段 3 确认后的各渠道版
   - 输出：各渠道发布许可
   - 决策：全渠道通过 → 进阶段 5；某渠道驳回 → 该渠道返回阶段 2
5. 阶段 5: 发布执行（根据渠道）
   - 需人工渠道（小红书 / 抖音 / 视频号）：创始人手机 App 发布
   - 可 API 渠道（公众号 / 头条 / 微博）：Agent 预留接口
   - 后续：运营 Agent 跟踪 24h/48h/7 天数据

**后置条件**: 内容发布完成，数据跟踪中
**涉及 SOP**: MKT-001

---

#### C3: 反馈处理

**触发**: 用户评论、私信、邮件、小程序反馈
**前置条件**: MKT-002 SOP 已加载
**主流程**:

1. 阶段 1: 反馈收集（自动）
   - 运营 Agent 全渠道监听
   - 来源：小红书评论 + 私信、抖音评论、公众号留言、邮件、小程序反馈
   - 输出：原始反馈清单（去重 / 去噪）
2. 阶段 2: 分级分类（串行）
   - 运营 Agent + 主 Agent
   - 输出：P0-P4 分级结果
3. 阶段 3: 响应处理（根据类型）
   - P0 紧急投诉 → 立即升级创始人
   - P1 功能故障 → 触发 DEV-002 Bug 修复 SOP
   - P2 体验建议 → 入需求池
   - P3 正向反馈 → 入素材库
   - P4 无效噪音 → 自动归档
4. 阶段 4: 闭环归档（串行）
   - 审计 Agent
   - 输出：归档记录 + 教训记录（如适用）

**后置条件**: 反馈处理完成，归档入库
**涉及 SOP**: MKT-002

---

### 场景组 D: 运营行政

#### D1: 财务月结

**触发**: 每月 1 日定时，或创始人手动触发
**前置条件**: OPS-001 SOP 已加载
**主流程**:

1. 阶段 1: 数据采集（自动）
   - 财务 Agent
   - 范围：收入 / 支出 / 账户余额 / 发票
   - 方式：手动录入 / 银行流水导入 / 发票 OCR
2. 阶段 2: 数据核对（串行）
   - 财务 Agent + 主 Agent
   - 检查：收入 + 支出 = 余额变动
   - 决策：无异常 → 进阶段 3；有异常 → 升级创始人
3. 阶段 3: 报表生成（串行）
   - 财务 Agent
   - 输出：损益表 / 现金流量表 / 税务准备清单
4. 阶段 4: 审计归档（串行）
   - 审计 Agent
   - 输出：归档记录

**后置条件**: 财务月结完成，报表可查看/导出
**涉及 SOP**: OPS-001

---

#### D2: 合规审计

**触发**: 项目发布前强制触发，或每季度定时
**前置条件**: OPS-002 SOP 已加载
**主流程**:

1. 阶段 1: 合规检查（自动）
   - 行政 Agent
   - 检查项：实体维护 / 合同管理 / 知识产权 / 隐私政策 / 税务节点 / 平台合规
   - 方式：本地规则引擎（90%），新型风险送大模型（10%）
2. 阶段 2: 风险识别（并行）
   - 行政 Agent + 法务 Agent
   - 输出：风险识别报告
3. 阶段 3: 整改执行（串行）
   - 主 Agent 调度
   - 输出：整改记录
   - 决策：完成 → 进阶段 4；无法整改 → 升级创始人
4. 阶段 4: 审计归档（串行）
   - 审计 Agent
   - 输出：合规审计报告

**关键规则**: 项目发布前强制触发，未通过主 Agent 拦截发布
**后置条件**: 合规审计完成，可发布或需整改
**涉及 SOP**: OPS-002

---

#### D3: 数据备份与归档

**触发**: 每周定时（周日 2:00）+ 重大操作后
**前置条件**: OPS-003 SOP 已加载
**主流程**:

1. 阶段 1: 备份执行（自动）
   - 运维 Agent
   - 范围：代码 / 文档 / 数据库 / 记忆 / 配置
2. 阶段 2: 完整性校验（串行）
   - 运维 Agent
   - 方式：本地哈希比对（MD5/SHA256）
3. 阶段 3: 异地同步（串行）
   - 运维 Agent
   - 方式：本地 rsync / 云存储 SDK
4. 阶段 4: 归档记录（串行）
   - 审计 Agent
   - 输出：备份归档记录

**关键规则**: 3-2-1 原则，季度恢复演练，全程 0 Token
**后置条件**: 备份完成，记录归档
**涉及 SOP**: OPS-003

---

#### D4: 合同与供应商管理

**触发**: 新合同签订，或到期前提醒
**前置条件**: OPS-004 SOP 已加载
**主流程**:

1. 阶段 1: 合同建档（串行）
   - 行政 Agent
   - 方式：本地 OCR 提取关键信息
2. 阶段 2: 履约监控（自动）
   - 行政 Agent
   - 规则：到期前 30/15/7 天提醒，本地日期计算
3. 阶段 3: 续约评估（串行）
   - 行政 Agent + 法务 Agent
   - 输出：续约建议（续 / 换 / 停）
   - 决策：创始人拍板
4. 阶段 4: 审计归档（串行）
   - 审计 Agent

**后置条件**: 合同管理闭环，提醒已设置
**涉及 SOP**: OPS-004

---

#### D5: 信息安全与资产

**触发**: 每月定时，或异常事件
**前置条件**: OPS-005 SOP 已加载
**主流程**:

1. 阶段 1: 资产盘点（自动）
   - 运维 Agent
   - 范围：硬件 / 软件 / 账号 / 密钥
2. 阶段 2: 安全检查（并行）
   - 运维 Agent + 安全 Agent
   - 检查项：密码强度 / 密钥过期 / 软件更新 / 访问日志 / 数据加密 / 备份可用
3. 阶段 3: 整改执行（串行）
   - 运维 Agent
   - 规则：仅提示不自动执行（防锁定）
4. 阶段 4: 审计归档（串行）
   - 审计 Agent

**关键规则**: 异常登录成功立即升级创始人，软件更新提示手动执行
**后置条件**: 安全检查完成，整改提示已生成
**涉及 SOP**: OPS-005

#### D6: 知识资产管理

**触发**: 新文档产生时自动触发，或创始人手动整理
**前置条件**: OPS-006 SOP 已加载
**主流程**:

1. 阶段 1: 捕获（自动）
   - 行政 Agent
   - 触发：新文档产生时
2. 阶段 2: 摘要（本地）
   - 本地小模型 3B
   - 输出：200 字摘要卡片
   - 时间：3 天后
3. 阶段 3: 图谱（本地）
   - 本地小模型 3B
   - 输出：实体 + 关系
   - 时间：7 天后
4. 阶段 4: 提炼（本地/云端）
   - 本地 3B 为主，复杂送大模型
   - 输出：知识晶体（500 字结构化）
   - 时间：30 天后
   - 规则：高频使用 >3 次 → 晶体；低频 → 备用
5. 阶段 5: 生命周期（本地）
   - 本地规则引擎
   - 30 天：提炼评估
   - 90 天：二次评估（晶体检查有效期 / 备用 → 归档 / 待归档 → 删除）
   - 180 天：最终清理（仅保留晶体，原文删除）

**后置条件**: 知识资产自动提炼、归档、清理
**涉及 SOP**: OPS-006

#### E2: 周期性回顾

**触发**: 日（可选）/ 周（建议）/ 月（必须）/ 季（必须）定时
**前置条件**: STR-002 SOP 已加载
**主流程**:

1. 阶段 1: 数据汇总（自动）
   - 秘书 Agent
   - 范围：任务 / 项目 / 能量 / Token / Agent 活动
2. 阶段 2: 分析（本地）
   - 本地小模型 3B
   - 输出：趋势分析报告
3. 阶段 3: 建议（本地/云端）
   - 主 Agent + 各 Agent
   - 输出：行动建议清单（P0/P1/P2）
4. 阶段 4: 回顾执行（人工）
   - 创始人决策：确认 / 调整 / 执行

**关键规则**: 连续跳过 2 次周回顾秘书 Agent 提醒，P0 建议必须执行
**后置条件**: 回顾完成，行动建议入库
**涉及 SOP**: STR-002

---

#### E3: 风险预警与应对

**触发**: 各 Agent 持续监控，自动触发
**前置条件**: STR-003 SOP 已加载
**主流程**:

1. 阶段 1: 风险监测（自动）
   - 各 Agent 持续监控
   - 来源：项目进度 / 财务健康 / 技术债务 / 用户反馈 / 创始人状态 / 外部环境
2. 阶段 2: 风险分级（本地）
   - 主 Agent + 本地规则
   - 输出：P0/P1/P2/P3
3. 阶段 3: 应对执行（根据等级）
   - P0: 立即中断所有工作，弹窗 + 声音 + 短信，创始人必须立即处理
   - P1: 自动调整资源，4 小时内创始人确认
   - P2: 自动调整，事后知情
   - P3: 记录跟踪，纳入下次回顾
4. 阶段 4: 复盘归档（串行）
   - 审计 Agent
   - 输出：复盘报告 + 教训记录 + 更新风险阈值

**后置条件**: 风险应对完成，归档入库
**涉及 SOP**: STR-003

## 5. 高频 vs 低频场景

|------|------|-----------|
| 每日 | A1 早晨启动、A2 任务执行、A3 状态记录、A4 晚间收尾 | **最高** —— 必须最顺手 |
