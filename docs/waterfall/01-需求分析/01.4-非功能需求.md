## 1. 性能需求

### 1.1 响应时间

| 场景 | 目标响应时间 | 可接受上限 | 测量方式 |
|------|-------------|-----------|---------|
| 仪表盘页面加载 | < 2 秒 | 5 秒 | 从点击到首屏渲染完成 |
| Agent 状态更新 | < 3 秒 | 10 秒 | 从 Agent 上报到 UI 刷新 |
| 意图解析（本地 3B） | < 2 秒 | 5 秒 | 从输入到返回分类结果 |
| 向量检索 | < 1 秒 | 3 秒 | 从查询到返回 Top-5 结果 |
| 日志面板刷新 | < 1 秒 | 3 秒 | 新日志产生到面板显示 |
| 任务结果生成（本地） | < 10 秒 | 30 秒 | 从阶段启动到输出完成 |
| 任务结果生成（云端） | < 30 秒 | 120 秒 | 从 API 调用到返回结果 |
| 文件生成（.docx/.xlsx） | < 5 秒 | 15 秒 | 从触发到文件保存完成 |

### 1.2 并发与负载

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 同时运行 Agent 数 | ≤ 4 个 | 创始人一人公司，无需高并发 |
| 同时执行任务数 | ≤ 2 个 | 主 Agent 调度，串行为主 |
| 历史任务存储 | ≥ 10,000 条 | 按 3 年每日 10 条估算 |
| 日志存储 | ≥ 100,000 条 | 按 3 年估算 |
| 知识晶体存储 | ≥ 5,000 个 | 按 3 年估算 |
| 向量数据规模 | ≤ 10GB | ChromaDB 本地存储 |
| SQLite 数据库 | ≤ 1GB | 结构化数据 |

### 1.3 资源占用

| 资源 | 目标值 | 上限 |
|------|--------|------|
| 内存（运行时） | ≤ 2GB | 4GB（含 Ollama 3B 模型） |
| 磁盘（安装包） | ≤ 500MB | 1GB |
| 磁盘（运行数据，热层） | ≤ 50GB | 50GB（硬限制） |
| CPU（空闲） | ≤ 5% | 10% |
| CPU（任务执行） | ≤ 50% | 80% |

## 2. 安全需求

### 2.1 数据安全

| 需求 ID | 需求描述 | 实现方式 | 验收标准 |
|---------|---------|---------|---------|
| NFR-SEC-001 | API Key 加密存储 | AES-256-GCM 加密，密钥派生自设备指纹 | 数据库中 API Key 不可明文读取 |
| NFR-SEC-002 | 敏感数据脱敏 | LLM 调用前自动脱敏：手机号→138****8888，邮箱→a***@example.com | 审计日志中 LLM 输入无敏感信息 |
| NFR-SEC-003 | 本地数据不出网 | 所有原始数据本地处理，仅脱敏后送云端 | 网络抓包验证无原始数据外发 |
| NFR-SEC-004 | 审计日志不可篡改 | 审计日志追加-only，哈希链校验 | 尝试修改日志文件触发告警 |
| NFR-SEC-005 | 备份加密 | 温层/冷层备份 AES 加密 | 备份文件不可无密钥读取 |

### 2.2 访问控制

| 需求 ID | 需求描述 | 实现方式 | 验收标准 |
|---------|---------|---------|---------|
| NFR-SEC-006 | Agent 权限隔离 | 每个 Agent 运行沙箱，权限白名单 | Agent A 无法访问 Agent B 的记忆 |
| NFR-SEC-007 | 工具调用权限校验 | 每次工具调用前检查 Agent 权限 | 无权限调用被拒绝并记录审计日志 |
| NFR-SEC-008 | 文件系统沙箱 | Agent 只能访问授权目录 | 尝试越界访问被拒绝 |
| NFR-SEC-009 | 代码执行沙箱 | 开发 Agent 代码在受限环境执行 | 危险操作（rm -rf /）被拦截 |

### 2.3 通信安全

| 需求 ID | 需求描述 | 实现方式 | 验收标准 |
|---------|---------|---------|---------|
| NFR-SEC-010 | API 通信加密 | HTTPS/TLS 1.3 | 抓包验证加密传输 |
| NFR-SEC-011 | 本地服务绑定 | 仅绑定 127.0.0.1 | 外部网络不可访问 |

## 3. 可用性需求

### 3.1 可靠性

| 需求 ID | 需求描述 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| NFR-AVA-001 | 系统启动成功率 | ≥ 99% | 连续 100 次启动失败 ≤ 1 次 |
| NFR-AVA-002 | 任务执行成功率 | ≥ 95% | 排除创始人主动取消的任务 |
| NFR-AVA-003 | 数据不丢失 | 100% | 异常崩溃后数据可恢复 |
| NFR-AVA-004 | 自动恢复 | 崩溃后 30 秒内自动重启 | 模拟崩溃测试 |

### 3.2 容错性

| 需求 ID | 需求描述 | 实现方式 | 验收标准 |
|---------|---------|---------|---------|
| NFR-AVA-005 | Agent 崩溃隔离 | 单个 Agent 崩溃不影响其他 Agent | 强制终止一个 Agent，其他继续运行 |
| NFR-AVA-006 | 任务断点续传 | 被抢占任务状态保存，恢复后从断点继续 | 模拟抢占后恢复，输出一致 |
| NFR-AVA-007 | 云端 API 降级 | 云端 API 失败时自动切换本地模型 | 断开网络后任务仍可完成（简化版） |
| NFR-AVA-008 | 数据库损坏恢复 | SQLite 损坏时自动从最近备份恢复 | 模拟数据库损坏，系统可恢复 |

### 3.3 易用性

| 需求 ID | 需求描述 | 目标用户 | 验收标准 |
|---------|---------|---------|---------|
| NFR-AVA-009 | 首次启动引导 | IT 小白创始人 | 首次启动有 5 步引导，5 分钟内完成配置 |
| NFR-AVA-010 | 错误提示友好 | IT 小白创始人 | 技术错误用中文解释，提供解决步骤 |
| NFR-AVA-011 | 操作反馈即时 | 所有操作 | 按钮点击后 100ms 内有视觉反馈 |
| NFR-AVA-012 | 帮助文档内嵌 | IT 小白创始人 | 每个页面有 ? 按钮，点击显示该页面功能说明 |
| NFR-AVA-013 | 快捷键支持 | 熟练用户 | 常用操作支持快捷键（Ctrl+N 新任务，Ctrl+R 回顾等） |

## 4. 可维护性需求

### 4.1 代码规范

| 需求 ID | 需求描述 | 标准 | 验收标准 |
|---------|---------|------|---------|
| NFR-MAINT-001 | Python 代码规范 | PEP 8 + 项目自定义规范 | flake8 检查零警告 |
| NFR-MAINT-002 | 类型注解 | 所有函数参数和返回值 | mypy 检查通过 |
| NFR-MAINT-003 | 文档字符串 | 所有公共函数和类 | docstring 覆盖率 100% |
| NFR-MAINT-004 | 单元测试覆盖率 | 核心逻辑 ≥ 80% | pytest --cov 报告 |

### 4.2 模块化

| 需求 ID | 需求描述 | 验收标准 |
|---------|---------|---------|
| NFR-MAINT-005 | Agent 可独立替换 | 替换一个 Agent 实现不影响其他 Agent |
| NFR-MAINT-006 | SOP 可独立添加 | 新增一个 SOP 只需添加模板文件，不改核心代码 |
| NFR-MAINT-007 | 前端可独立替换 | 前端框架可从 NiceGUI 切换为 Flutter，后端接口不变 |
| NFR-MAINT-008 | 数据库可独立替换 | SQLite 可切换为 PostgreSQL，只需改配置 |

### 4.3 日志与监控

| 需求 ID | 需求描述 | 验收标准 |
|---------|---------|---------|
| NFR-MAINT-009 | 结构化日志 | 所有日志为 JSON 格式，含时间、级别、模块、消息 |
| NFR-MAINT-010 | 日志分级 | DEBUG/INFO/WARNING/ERROR/CRITICAL，生产环境默认 INFO |
| NFR-MAINT-011 | 性能监控 | 关键操作（LLM 调用、数据库查询、文件操作）记录耗时 |
