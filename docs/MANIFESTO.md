## 我们相信什么

### 1. 一人公司，完整闭环

S-O-P 不是工具箱的拼凑，而是一个**操作系统**——管理三件事：公司日常、三个项目（心域探险 / 心栖陪伴 / 心理歌曲）、你自己的状态。

- **端到端工作流**：从任务下发、Agent 协作、结果输出，一个平台完成
- **文档直接生成**：.docx / .xlsx / .pptx / .pdf，工作流输出即交付物
- **私有化部署**：前端 NiceGUI（暗色主题默认），后端 Python + SQLite + ChromaDB，完全掌控自己的数据和隐私

### 2. 数据主权在你手里

- **本地优先**：所有文档处理、图片理解、向量检索都在你的机器上完成（Ollama qwen2.5:3b + nomic-embed-text）
- **加密存储**：API Key 和个人信息通过 RSA + AES 端到端加密存储
- **脱敏机制**：LLM 只能看到脱敏后的数据，原始数据不出本地
- **不上传**：你的记忆、你的文件、你的操作记录，不会上传到任何人的服务器
- **分层存储**：热层（50GB SSD 活跃数据）→ 温层（256GB 外接 SSD 备份）→ 冷层（1TB 机械归档）

### 3. 透明不是口号，而是设计

- **工具调用全审计**：每次任务调了什么工具、传了什么参数、返回什么结果，全部实时可查
- **过程可追溯**：所有 Agent 操作有迹可循，拒绝黑箱
- **状态可视化**：
  - **纵向**：单个任务全过程可视（哪个 Agent 做了什么、到什么阶段、还需做什么）
  - **横向**：整个公司各项业务进展情况
- **Token 成本透明**：月度预算 ¥300，每次操作消耗实时显示，80% 预警，100% 熔断

### 4. Agent 可控，而非自治

12 个预设 Agent，每个在**笼子**里运行：

| Agent | 核心职责 | 权限边界 |
|-------|---------|---------|
| 主 Agent | 接收命令、解析意图、选择 SOP、组建团队 | 不能替代你决策 |
| 审计 Agent | 代码审计、需求审计、SOP 补丁确认 | 技术问题裁决权大于你 |
| 秘书 Agent | 日程管理、提醒、记录、通知 | 访问你的私人日程 |
| 产品经理 Agent | PRD 撰写、需求拆解、用户故事 | 不能决定发布 |
| 架构师 Agent | 技术选型、架构设计、接口定义 | 不能写实现代码 |
| 开发 Agent | 写代码、单元测试、Git 提交 | 不能部署生产、不能访问 Vault |
| 测试 Agent | 测试用例、回归测试、Bug 报告 | 不能修改代码 |
| 营销 Agent | 选题、渠道策略、数据分析 | 不能发布内容 |
| 内容 Agent | 母酱生产、文案、脚本 | 不能决定发布 |
| 运营 Agent | 反馈收集、社区维护、数据跟踪 | 不能修改产品 |
| 财务 Agent | 记账、报表、税务准备 | 不能决策投资 |
| 行政 Agent | 合同、合规、资产、档案 | 不能决策业务方向 |

**关键规则**：
- **唤醒可控**：只有你的指令或你设定的定时任务才会唤醒 Agent，不会自我唤醒
- **权限隔离**：多 Agent 协同但各司其职，操作范围受限于你授予的权限
- **记忆隔离**：每个 Agent 专属记忆空间（ChromaDB 集合），项目级隔离，跨项目需主 Agent 授权
- **人类在环**：关键决策点必须经你确认，Agent 不擅自行动

### 5. 自我进化，但受你监督

- **自我审计**：Agent 能审查自己的操作，发现异常主动报告
- **技能自扩展**：支持动态创建和加载 SKILLS，能力边界由你定义
- **记忆自组织**：长期记忆自动归纳、去重、升级，但你可以随时查看和修正
- **教训提取**：同类型踩坑 ≥3 次 → 触发教训归集 → 审计 Agent 提炼 SOP 补丁 → 自动生效（你事后知情）

### 6. 顺手，不是功能堆砌

创始人定义的"顺手"四标准：

| 标准 | 含义 | 设计体现 |
|------|------|---------|
| **有序** | Agent 很清楚现在要干什么 | 12 个 SOP 模板，按工作性质执行，先讨论再干活 |
| **可视** | 全过程可视可溯源 | 纵向（单任务全流程）+ 横向（全公司业务进展） |
| **有层次** | 你只向主 Agent 发布命令 | 主 Agent 自行组织各专业 Agent 协同 |
| **有驱动** | 你自己也有任务列表和日程 | 秘书 Agent 记录私人日程 + 项目依赖（带时间窗口和提醒） |

## 决策权分层（你 vs 审计 Agent）

| 场景 | 决策权 | 处理方式 |
|------|--------|---------|
| 业务方向（做什么 / 不做什么 / 优先级） | **你** | 你拍板，审计 Agent 记录 |
| 技术方案（用什么技术 / 架构设计） | **审计 Agent** | 审计 Agent 裁决，你可质疑但需技术理由 |
| 质量标准（致命 / 警告 / 建议判定） | **审计 Agent** | 依据预定义标准 |
| 发布时间（什么时候上线） | **你** | 你决定，审计 Agent 评估风险后建议 |
| 资源分配（Agent 时间 / 优先级调整） | **主 Agent** | 根据规则自动调度，你可见可干预 |
| SOP 补丁生效（技术类） | **审计 Agent** | 自动生效，你事后知情 |
| 争议处理 | **讨论决定** | 业务问题你大，技术问题 Agent 大 |
