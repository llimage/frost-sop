## 3. 代码覆盖率详细分析

### 3.1 总体覆盖率

```
总语句数:   8,477
已覆盖:     ~4,973 (推测)
未覆盖:     3,226 语句 + 部分分支
覆盖率:     58.64%
```

### 3.2 覆盖率评级对照表

| 等级 | 覆盖率范围 | 本项目 |
|------|-----------|--------|
| A+ | ≥90% | ❌ |
| A | ≥80% | ❌ |
| B+ | ≥70% | ❌ |
| B | ≥60% | ❌ |
| C | ≥50% | ❌ (58.64% → 落在C级下限) |
| **D** | **<60%** | **✅ 本项目: 58.64%** |

**诚实的结论: 代码覆盖率评级为 D 级**，远低于行业最佳实践（A级≥80%）。

### 3.3 覆盖率短板分析（诚实评估）

覆盖率低的主要原因：
1. **大量边界路径未测试** — 错误处理分支、异常恢复流程等
2. **NiceGUI驾驶舱代码(app.py)未纳入测试范围** — 483行UI代码零测试
3. **Next.js前端代码未纳入** — frontend/目录完全无测试
4. **数据库迁移脚本无测试** — 迁移逻辑的正确性未验证
5. **集成场景覆盖不足** — 多Agent并发、长时间运行的内存泄漏等

## 4. 变异测试报告 (#45)

### 4.1 执行摘要

| 指标 | 数值 |
|------|------|
| 工具 | cosmic-ray v8.4.6 |
| 测试目标 | `core/monitor.py` (仅1个模块) |
| 变异算子 | ReplaceBinaryOperator (Sub→Add/Mul/Div/FloorDiv/Mod/Pow/RShift) |
| Kill Rate | **52.44%** |
| 评级 | **F** |

### 4.2 变异详情

| 变异算子 | 结果 |
|----------|------|
| ReplaceBinaryOperator_Sub_Add | SURVIVED |
| ReplaceBinaryOperator_Sub_Mul | SURVIVED |
| ReplaceBinaryOperator_Sub_Div | SURVIVED |
| ReplaceBinaryOperator_Sub_FloorDiv | SURVIVED |
| ReplaceBinaryOperator_Sub_Mod | SURVIVED |
| ReplaceBinaryOperator_Sub_Pow | KILLED |

### 4.3 诚实声明

- **变异测试仅覆盖1个模块** — 未达到任何评级标准
- 52.44%的kill rate意味着：给定模块中近一半的算术变异未被测试捕获
- 存活变异主要是 `perf_counter()` 计时相关的替换（减→加/乘/除等），现有测试只验证函数返回值，不验证具体时间值
- **未对其他核心模块（db.py, orchestration.py, assemble.py, event_bus.py等）进行变异测试**
